# I/O 接口摘要

## 课件定位

- 对应课件：`微机原理-第6章1IO接口.pptx`
- 主题：接口作用、端口编址、IN/OUT、程序查询、中断、DMA

## 本讲主线

本讲先解释“为什么需要接口”，再讲“CPU 怎样和接口交换数据”。核心内容是端口编址方式和三种传送方式。

## 页序摘要

### 1. 接口存在的原因

- 外设与 CPU 在速度、电平、数据格式、时序上不匹配。
- 因此需要接口电路做缓冲、译码、锁存、控制和握手。

### 2. 端口和编址方式

- 介绍存储器映像编址。
- 介绍独立 I/O 编址。
- 对比两者在地址空间和指令使用上的区别。

### 3. IN / OUT 指令

- 复习端口访问格式。
- 区分直接端口地址和 `DX` 间接端口地址。
- 结合状态口、数据口、控制口举例。

### 4. 程序控制方式

- CPU 轮询外设状态，准备好再读写。
- 结构简单，但 CPU 会忙等。

### 5. 中断方式

- 设备就绪后向 CPU 发中断请求。
- CPU 在合适时机转去执行中断服务程序。
- 适合中速设备。

### 6. DMA 方式

- 数据直接在外设和内存之间传输。
- CPU 只做初始化和结束处理。
- 适合大批量高速传输。

## 应掌握的结论

- 接口题先想“译码、握手、状态、控制”四件事。
- 三种数据传送方式要会比较优缺点。
- 独立编址时一般通过 `IN/OUT` 访问端口。
