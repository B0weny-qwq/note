# 信源编码和信道编码的区别

## 两者为什么容易混

因为它们都叫“编码”，都把原始数据变成另一种表示方式。但目标完全不同。

## 核心区别

`信源编码`
: 尽量删掉冗余，让数据更短。

`信道编码`
: 主动增加冗余，让数据更不容易错。

## 详细例子

原始比特串：

```text
1010011
```

如果做信源编码：

- 目标是变短，比如从 7 位压到 5 位。
- 前提是消息中有可利用的统计冗余。

如果做信道编码：

- 目标是变稳，比如在末尾加一个偶校验位：

```text
10100110
```

- 这样传输后即使出错，也有机会发现甚至纠正。

## 生活类比

旅行打包：

- 信源编码像压缩衣服，箱子更省空间。
- 信道编码像给易碎品包泡泡膜，体积更大但更安全。

## 常见算法例子

信源编码：

- Huffman 编码
- Shannon 编码
- 算术编码

信道编码：

- 线性分组码
- 汉明码
- 循环码
- 卷积码

## 易错点

- “压缩比高”和“抗干扰强”通常不是同一方向。
- 同一系统里通常两类编码都要用，但位置不同。

## 注意点

- 通信系统设计里，先想有效性，再想可靠性，最后才是具体实现。
