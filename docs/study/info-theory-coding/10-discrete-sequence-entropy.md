# 离散序列信源的熵

## 为什么序列信源单独讲

真实通信里很少只发一个孤立符号，更多是连续符号序列。  
所以要研究“一个序列整体有多少平均信息”。

## 无记忆离散序列

如果每个符号独立同分布，那么长度为 `n` 的序列总熵近似就是单符号熵乘以 `n`。

```text
H(X1, X2, ... , Xn) = nH(X)
```

## 有记忆离散序列

如果前后符号相关，后一个符号更容易预测，那么平均每个符号提供的信息会下降。

这就引出了熵率的概念。

## 熵率

熵率就是“每个符号平均带来的信息量”。

可以理解成：

```text
总熵 / 序列长度
```

## 详细例子

中文文本中，“中”后面接“国”的概率远高于随机。  
说明序列里有结构和记忆，后一个符号不是完全独立的，所以整体平均信息量会比“每个字完全随机”更低。

## 生活类比

你猜歌歌词时，听到前半句，后半句更容易猜。  
这说明歌词序列有记忆性，也说明冗余存在。

## 易错点

- 序列越长，不代表“每个符号的信息量”越大。

## 注意点

- 信源存在相关性，才会出现可以利用的压缩空间。
