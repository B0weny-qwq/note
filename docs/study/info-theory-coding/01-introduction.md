# 绪论

## 背景

信息论与编码关心三个核心问题：信息如何度量、如何高效压缩、如何在有噪声信道中可靠传输。后续各章基本都围绕这三件事展开。

## 课程主线

可以把整门课看成一条链：

```text
信源建模 -> 信息量与熵 -> 信道模型与容量 -> 信源编码 -> 信道编码
```

其中：

- 信源部分回答“信息有多大、能压到多短”。
- 信道部分回答“链路最多能传多快”。
- 编码部分回答“怎样逼近这些理论极限”。

## 研究对象

课程把通信系统抽象成：

- 信源
- 编码器
- 信道
- 解码器
- 信宿

信息论更关注模型、概率和极限，编码理论更关注可实现的方法。

## 三个基本量

后面最常出现的三个量：

- 熵 `H(X)`：信源平均不确定度。
- 互信息 `I(X;Y)`：输入和输出共享的信息量。
- 容量 `C`：信道可支持的最大平均信息传输率。

## 两类编码问题

- 信源编码：减少冗余，提高压缩效率。
- 信道编码：增加监督信息，提高抗差错能力。

两者方向相反，但目标一致，都是优化信息传输系统。

## 学习时的固定问题

每一章都可以追问：

1. 数学模型是什么。
2. 度量量是什么。
3. 最优目标是什么。
4. 达到最优的条件是什么。
5. 工程实现用什么编码方法逼近它。
