# 信息位与监督位

## 定义

`信息位`
: 真正承载原始消息内容的位。

`监督位`
: 为了检错或纠错而额外加入的位，也常叫冗余位、校验位。

## 基本关系

如果：

- 信息位数是 `k`
- 监督位数是 `r`

那么码长通常是：

```text
n = k + r
```

## 详细例子

原始消息：

```text
1011
```

如果再加一个偶校验位，变成：

```text
10110
```

这里：

- `1011` 是信息位
- 最后那个 `0` 是监督位

## 生活类比

像快递箱里的发票和防伪标签：

- 真正的商品是信息位
- 为了核对和追溯多出来的标签，就是监督位

## 易错点

- 监督位不是“没用的信息”，它是为了可靠性付出的成本。

## 注意点

- 监督位越多，不一定永远最好，因为码率会下降、复杂度会上升。
