# 信息熵

## 定义

熵是信源平均每次输出所提供的信息量，也叫平均不确定度。

公式：

```text
H(X) = -Σ p(x) log p(x)
```

如果对数底数为 2，单位是 `bit/符号`。

## 直觉理解

熵越大，说明越难提前猜中。  
熵越小，说明越容易预测。

## 详细例子

### 例 1：均匀二元信源

如果 `P(0)=1/2, P(1)=1/2`：

```text
H(X) = 1 bit
```

这是二元信源的最大熵情况。

### 例 2：偏置二元信源

如果 `P(0)=0.9, P(1)=0.1`：

熵会明显小于 1，因为大多数时候你都能猜到输出是 `0`。

### 例 3：确定性信源

如果某个符号必然发生：

```text
P(x)=1
```

那么：

```text
H(X)=0
```

因为完全没有不确定性。

## 生活类比

如果老师每次上课都点同一个人回答问题，那你几乎能猜到，熵很小。  
如果老师随机点名，谁都有可能被点到，熵就大。

## 最大熵直觉

在符号个数固定时，等概率分布最乱、最难猜，所以熵最大。

## 易错点

- 熵不是“越大越好”，它只表示不确定性更高。
- 熵是平均量，不是某一次事件的意外程度。

## 注意点

- 熵是无失真压缩的理论核心。
- 后面平均码长的下界就是围绕熵展开的。
