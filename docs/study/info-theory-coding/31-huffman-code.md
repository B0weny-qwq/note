# Huffman 编码

## 定义

Huffman 编码是对单符号离散无记忆信源构造最优前缀码的经典方法。

## 构造步骤

1. 把所有符号按概率从小到大排序。
2. 取最小的两个概率合并成一个新节点。
3. 继续排序并重复。
4. 最后从树根回填 0/1 得到各符号码字。

## 详细例子

如果 4 个符号概率分别为：

```text
0.4, 0.3, 0.2, 0.1
```

先合并 `0.1` 和 `0.2`，再与其它节点逐步合并，最后生成一棵 Huffman 树。

高概率符号离根更近，码字更短。

## 为什么它重要

它能得到平均码长最短的前缀码，因此在教材和工程里都非常核心。

## 生活类比

像商场给高频商品分配离门口更近的位置，低频商品放更远。  
常用的路径更短，总体平均行走距离最小。

## 易错点

- Huffman 编码结果不唯一，但平均码长可相同。
- 构树时没保持小概率优先，会直接错。

## 注意点

- 题目一旦要求具体码字，最好把树完整画出来。
