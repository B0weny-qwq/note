# 算术编码

## 定义

算术编码不是给每个符号单独分配固定码字，而是把整个符号序列映射到一个区间。

## 核心思想

初始区间设为 `[0,1)`。  
每读入一个符号，就按该符号概率把当前区间再切一刀，只保留对应子区间。  
最后只要传输这个最终小区间中的任意一个数，就能代表整个序列。

## 详细例子

如果字母表是：

- `A` 概率 0.6
- `B` 概率 0.4

那么 `[0,1)` 可先切成：

- `A -> [0, 0.6)`
- `B -> [0.6, 1)`

如果序列是 `AB`，就在 `A` 的区间内再按同样规则切第二次。

## 为什么它强

因为它不是“每个符号必须凑整数个 bit”，所以对长序列能非常接近理论熵极限。

## 生活类比

像地图上不断缩小定位范围：

- 先知道你在某个城市
- 再知道你在某个区
- 再知道你在某条街

最后一个很小的区域就足以唯一确定目标。

## 易错点

- 算术编码不是把符号简单拼接成小数，而是每一步都按概率重新细分区间。

## 注意点

- 课件强调它在长序列下编码效率很高，且可接近熵极限。
