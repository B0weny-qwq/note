# 怎么做存储器扩展地址范围题

## 这类题在考什么

题目通常给你一张连接图，问：

```text
这片存储器的地址范围是多少
容量是多少
有没有地址重叠
```

核心是读出地址线分工。图只是把片内地址线和片选线画出来。

## 必背关系

容量和地址线：

```math
2^n=\text{地址单元数}
```

结束地址：

```math
\text{结束地址}=\text{起始地址}+\text{容量}-1
```

常用容量换算：

```math
1\mathrm{KB}=2^{10}=400\mathrm{H}
```

```math
2\mathrm{KB}=800\mathrm{H}
```

```math
4\mathrm{KB}=1000\mathrm{H}
```

```math
8\mathrm{KB}=2000\mathrm{H}
```

```math
16\mathrm{KB}=4000\mathrm{H}
```

```math
32\mathrm{KB}=8000\mathrm{H}
```

## 通用做题步骤

### 第一步：先找片内地址线

芯片容量决定片内地址线数量。

例如 $8\mathrm{K}\times8$：

```math
8\mathrm{K}=2^{13}
```

所以片内地址线有 13 根，通常是芯片 `A12~A0`。

### 第二步：再找片选条件

高位地址线送到译码器或门电路，用来生成 `CS#`。

这些位在同一片芯片内部保持固定，用来决定这片芯片是否被选中。

### 第三步：写固定高位和变化低位

做地址范围时，把地址写成两部分：

```text
固定高位 | 变化低位
```

固定高位来自片选条件，变化低位来自片内地址线。

### 第四步：算起止地址

片内地址线全为 0，是起始地址。

片内地址线全为 1，是结束地址。

也可以直接用：

```math
\text{结束地址}=\text{起始地址}+\text{容量}-1
```

## 例题 1：4KB 从 8000H 开始

已知容量为 $4\mathrm{KB}$，起始地址为 `8000H`。

容量换成十六进制长度：

```math
4\mathrm{KB}=1000\mathrm{H}
```

结束地址：

```math
8000\mathrm{H}+1000\mathrm{H}-1=8FFF\mathrm{H}
```

答案：

```text
8000H~8FFFH
```

考点：结束地址必须减 1。

## 例题 2：64K×8 用 8K×8 芯片扩展

目标容量：

```math
64\mathrm{K}\times8
```

单片容量：

```math
8\mathrm{K}\times8
```

需要芯片数：

```math
\frac{64\mathrm{K}}{8\mathrm{K}}=8
```

目标存储器有：

```math
64\mathrm{K}=2^{16}
```

所以系统内这个存储区需要 `A15~A0` 共 16 根地址线。

单片芯片有：

```math
8\mathrm{K}=2^{13}
```

所以芯片内部接 `A12~A0`。

剩余高位：

```text
A15 A14 A13
```

接 74LS138 的 `CBA`，输出 `Y0#~Y7#` 分别选 8 片。

若从 `0000H` 开始，地址范围为：

| 芯片 | 片选 | 地址范围 |
| --- | --- | --- |
| 0 | `Y0#` | `0000H~1FFFH` |
| 1 | `Y1#` | `2000H~3FFFH` |
| 2 | `Y2#` | `4000H~5FFFH` |
| 3 | `Y3#` | `6000H~7FFFH` |
| 4 | `Y4#` | `8000H~9FFFH` |
| 5 | `Y5#` | `A000H~BFFFH` |
| 6 | `Y6#` | `C000H~DFFFH` |
| 7 | `Y7#` | `E000H~FFFFH` |

考点：`A15~A13` 只负责选片，`A12~A0` 才进入每片芯片。

## 例题 3：74LS138 连线反推 98000H~9FFFFH

PPT 页末练习给出 RAM 与 74LS138 的连接，答案为：

```text
98000H~9FFFFH，容量 32KB
```

按地址范围反推可以看出固定高位：

```math
98000\mathrm{H}\div 8000\mathrm{H}=13\mathrm{H}
```

因为 $32\mathrm{KB}=8000\mathrm{H}$，低 15 位在片内变化。

`13H` 写成 5 位二进制：

```math
13\mathrm{H}=10011_2
```

所以高位固定为：

```text
A19 A18 A17 A16 A15 = 1 0 0 1 1
```

片内地址线为：

```text
A14~A0
```

容量：

```math
2^{15}=32768=32\mathrm{KB}
```

结束地址：

```math
98000\mathrm{H}+8000\mathrm{H}-1=9FFFF\mathrm{H}
```

考点：看见 32KB，就知道低 15 位变化；其余高位由译码器固定。

## 例题 4：EPROM 2732 和 RAM 6116 地址范围

PPT 末尾例题给出：

- EPROM 2732：$4\mathrm{K}\times8$
- SRAM 6116：$2\mathrm{K}\times8$

2732 每片容量：

```math
4\mathrm{KB}=1000\mathrm{H}
```

所以两片连续 EPROM 地址为：

```text
8000H~8FFFH
9000H~9FFFH
```

6116 每片容量：

```math
2\mathrm{KB}=800\mathrm{H}
```

所以两片连续 RAM 地址为：

```text
A000H~A7FFH
A800H~AFFFH
```

考点：先看单片容量，再按连续地址块往后排。

## 例题 5：27128 全译码到 1C000H~1FFFFH

27128 容量为：

```math
16\mathrm{K}\times8=4000\mathrm{H}
```

片内地址线：

```text
A13~A0
```

题目要求地址范围：

```text
1C000H~1FFFFH
```

因为容量是 `4000H`，低 14 位变化，高位固定。

起始地址二进制分组：

```text
1C000H = 0001 1100 0000 0000 0000B
```

低 14 位给芯片内部，固定高位为：

```text
A19~A14 = 000111
```

用 74LS138 时，可以让 `A17 A16 A15 = 011` 选中 `Y3#`，再把 `A19=0`、`A18=0`、`A14=1` 放入使能条件。

考点：全译码必须让所有固定高位都参与片选，否则会出现镜像地址。

## 例题 6：2732 部分译码从 10000H 开始

目标：用 4 片 2732 组成 $16\mathrm{K}\times8$，起始地址 `10000H`。

2732 单片容量：

```math
4\mathrm{KB}=1000\mathrm{H}
```

4 片连续地址：

| 芯片 | 地址范围 |
| --- | --- |
| 2732(0) | `10000H~10FFFH` |
| 2732(1) | `11000H~11FFFH` |
| 2732(2) | `12000H~12FFFH` |
| 2732(3) | `13000H~13FFFH` |

片内地址线：

```text
A11~A0
```

用于选 4 片的地址线：

```text
A13 A12
```

PPT 的部分译码方案让 `A16~A12` 参与译码，`A19~A17` 不参与。

所以合法窗口可以写成：

```text
10000H~13FFFH
```

但镜像窗口也存在：

```text
30000H~33FFFH
50000H~53FFFH
70000H~73FFFH
90000H~93FFFH
B0000H~B3FFFH
D0000H~D3FFFH
F0000H~F3FFFH
```

考点：部分译码题要说明未参与译码的高位会产生镜像。

## 易错点

- 把容量 `4KB` 写成结束地址 `4000H`，忘记加起始地址。
- 结束地址没有减 1。
- 把片内地址线和片选地址线混在一起。
- 用 74LS138 时忘记输出低有效。
- 部分译码题只写主地址范围，不写镜像风险。
