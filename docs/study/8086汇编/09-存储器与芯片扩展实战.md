# 存储器与芯片扩展实战

## 这章考什么

第 5 章的主线很明确：先认识 RAM、ROM 和典型芯片，再把芯片接到 CPU 总线上。

考试重点集中在四类题：

- 根据芯片容量判断地址线、数据线和芯片数量。
- 根据片选译码电路求地址范围。
- 按目标容量设计位扩展、字扩展或字位扩展。
- 分析 8086 系统的奇偶存储体、`A0` 和 `BHE#` 片选逻辑。

做题时不要先背芯片型号。先抓三个量：

```text
目标容量、单片容量、CPU 数据总线宽度
```

## 存储器分类

### 内存和外存

内存也叫主存，CPU 可以直接访问。8086 通过 20 根地址线访问主存，最大寻址空间为：

```math
2^{20}=1\ \mathrm{MB}
```

外存不能直接和 CPU 交换数据，要先通过接口把数据送入内存。硬盘、U 盘、光盘、磁带都属于外存。

### RAM 和 ROM

RAM 可读可写，断电后内容通常丢失。

常见 RAM：

| 类型 | 特点 | 常见用途 |
| --- | --- | --- |
| SRAM | 速度快，不需刷新，容量较小 | Cache、小容量高速缓冲 |
| DRAM | 容量大、价格低，需要刷新 | PC 主存 |
| PSRAM | 接口像 SRAM，内部接近 DRAM | 早期移动设备 |
| SDRAM/DDR/LPDDR | 同步、突发传输、低功耗版本 | 主存、嵌入式内存 |

ROM 断电后内容不丢失。

常见 ROM：

| 类型 | 是否可改写 | 典型用途 |
| --- | --- | --- |
| Mask ROM | 出厂固定 | 大批量固化程序 |
| PROM | 只能写一次 | 一次性程序固化 |
| EPROM | 紫外线擦除后重写 | 实验、仪器程序 |
| EEPROM | 电擦写 | 参数保存、配置存储 |
| Flash | 电擦写，容量大 | BIOS、U 盘、SSD |

## 典型芯片要记什么

题目给芯片型号时，考点通常落在容量和控制脚。

| 芯片 | 容量 | 地址线 | 数据线 | 做题关注点 |
| --- | --- | --- | --- | --- |
| 2114 SRAM | $1\mathrm{K}\times4$ | 10 根 | 4 根 | 常用于字位扩展例题 |
| 6264 SRAM | $8\mathrm{K}\times8$ | 13 根 | 8 根 | 有两个片选端，常用于 8086 RAM |
| 62256 SRAM | $32\mathrm{K}\times8$ | 15 根 | 8 根 | 32KB 芯片，地址线到 `A14` |
| 2716 EPROM | $2\mathrm{K}\times8$ | 11 根 | 8 根 | 小容量 EPROM |
| 2732 EPROM | $4\mathrm{K}\times8$ | 12 根 | 8 根 | PPT 部分译码和 8086 ROM 例题 |
| 2764 EPROM | $8\mathrm{K}\times8$ | 13 根 | 8 根 | PPT 线选法例题 |
| 27128 EPROM | $16\mathrm{K}\times8$ | 14 根 | 8 根 | PPT 全译码例题 |

地址线数量按下面关系算：

```math
2^n=\text{芯片可寻址单元数}
```

例如 6264 是 $8\mathrm{K}\times8$：

```math
8\mathrm{K}=8\times 2^{10}=2^{13}
```

所以 6264 需要 13 根地址线。

## CPU 和存储器连接看四组线

### 地址线

低位地址线接芯片地址脚，完成片内寻址。

高位地址线送入译码器，生成片选信号。

```text
低位地址线：找芯片内部哪个单元
高位地址线：找哪一片芯片
```

### 数据线

数据线宽度要和 CPU 匹配。

8086 数据总线为 16 位。若使用 $8$ 位芯片，必须分成低 8 位存储体和高 8 位存储体。

### 控制线

RAM 要能读也能写，常见控制连接：

```text
OE# 接读控制
WE# 接写控制
CS# 接片选
```

ROM 只读，通常只接读允许和片选，不接写控制。

### 片选线

片选信号决定 CPU 本次访问哪片芯片。

片选设计出错会带来两类问题：

- 多片芯片同时驱动数据总线，发生总线冲突。
- 一个存储单元对应多个地址，出现地址重叠或镜像。

## 三种扩展方式

### 位扩展

单片数据位数不够时做位扩展。

例：用 $64\mathrm{K}\times1$ 芯片组成 $64\mathrm{K}\times8$ 存储器。

计算：

```math
\frac{8}{1}=8
```

所以要 8 片。8 片地址线并联，控制线并联，每片负责 1 根数据线。

### 字扩展

单片字数不够、数据位数够时做字扩展。

例：用 $16\mathrm{K}\times8$ 芯片组成 $64\mathrm{K}\times8$ 存储器。

计算：

```math
\frac{64\mathrm{K}}{16\mathrm{K}}=4
```

所以要 4 片。每片内部地址线并联，高位地址线译码产生 4 个片选。

### 字位扩展

字数和位数都不够时，先做位扩展，再做字扩展。

例：用 $1\mathrm{K}\times4$ 的 2114 构成 $4\mathrm{K}\times8$ 存储器。

位数方向：

```math
\frac{8}{4}=2
```

字数方向：

```math
\frac{4\mathrm{K}}{1\mathrm{K}}=4
```

总片数：

```math
2\times4=8
```

做题顺序：

1. 2 片 2114 并成一组，得到 $1\mathrm{K}\times8$。
2. 4 组再做字扩展，得到 $4\mathrm{K}\times8$。
3. `A9~A0` 接片内地址线，`A11~A10` 译码选 4 组。

## 片选译码三种方法

### 线选法

线选法直接用某一位高位地址线做片选。

优点是电路简单。缺点是未参与译码的高位地址会形成镜像地址，地址空间浪费大。

### 全译码法

全译码法让所有需要固定的高位地址都参加译码。

优点是每个存储单元地址唯一。缺点是译码电路更复杂。

### 部分译码法

部分译码法只选一部分高位地址参与译码。

优点是电路比全译码简单。缺点是会出现重叠地址。小系统可以用，地址空间紧张的系统不要随便用。

## 74LS138 怎么用

74LS138 是 3-8 译码器。它有 3 个编码输入 `CBA`，8 个低有效输出 `Y0#~Y7#`。

使能条件：

```text
G1 = 1
G2A# = 0
G2B# = 0
```

输出规律：

```text
CBA = 000 -> Y0# = 0
CBA = 001 -> Y1# = 0
CBA = 010 -> Y2# = 0
...
CBA = 111 -> Y7# = 0
```

考题常把高位地址线接到 `CBA`，再用某个 `Y#` 接芯片 `CS#`。

## 地址范围题固定流程

地址范围题按 4 步做：

1. 看芯片容量，算片内地址线数量。
2. 找哪些地址线接芯片地址脚，这些位在片内变化。
3. 找哪些高位地址线参与片选，这些位决定起始地址。
4. 用容量求结束地址。

结束地址公式：

```math
\text{结束地址}=\text{起始地址}+\text{容量}-1
```

例：某片容量为 $8\mathrm{KB}$，起始地址为 `8000H`：

```math
8000\mathrm{H}+2000\mathrm{H}-1=9FFF\mathrm{H}
```

地址范围为：

```text
8000H~9FFFH
```

## 8086 奇偶存储体

8086 有 16 位数据总线，但地址按字节编号。

使用 $8$ 位存储芯片组成 16 位存储器时，要分两组：

- 偶地址存储体：接 `D7~D0`。
- 奇地址存储体：接 `D15~D8`。

片选由 `A0` 和 `BHE#` 配合：

| 访问类型 | `A0` | `BHE#` | 选中的存储体 |
| --- | --- | --- | --- |
| 偶地址字节 | 0 | 1 | 偶体 |
| 奇地址字节 | 1 | 0 | 奇体 |
| 偶地址字 | 0 | 0 | 奇体和偶体同时选中 |
| 奇地址字 | 1 | 0 后接下一周期 | 需要两个总线周期 |

记法：

```text
A0 管低 8 位偶体，BHE# 管高 8 位奇体。
```

## 章末检查清单

- 芯片容量是否换成了地址单元数。
- 片内地址线和片选地址线是否分清。
- 扩展方式是否先位扩展、再字扩展。
- 地址范围是否用了“起始地址 + 容量 - 1”。
- 部分译码题是否写出镜像地址。
- 8086 题是否考虑了 `A0` 和 `BHE#`。
