# 怎么区分查询、中断和 DMA

## 这类题在考什么

这类题通常考：

```text
设备和 CPU 之间的数据传输到底属于哪一类方式
```

常见就是选择题、判断题。

## 先记公式

最稳的三句话：

```text
查询：CPU 一直主动问
中断：设备好了再通知 CPU
DMA：由 DMA 控制器直接搬大块数据
```

## 做题步骤

### 第一步：看 CPU 是不是一直轮询状态

如果程序不断读状态口等待，  
那通常是查询方式。

### 第二步：看设备是不是“准备好再打断 CPU”

如果描述是“设备发中断请求”，  
那就是中断方式。

### 第三步：看数据是不是不经过 CPU 一条条搬

如果题目强调大块数据由专门控制器搬运，  
那就是 DMA。

### 第四步：再结合场景判断

- 简单低速设备常见查询
- 响应更及时常见中断
- 大块高速传输常见 DMA

## 最简单例子

如果题目说：

```text
CPU 反复读取设备状态寄存器，
直到 ready 位置 1 再读数据
```

那就是：

```text
查询方式
```

如果题目说：

```text
磁盘大块数据直接送入内存，
CPU 只负责初始化和结束处理
```

那就是：

```text
DMA
```

## 易错点

- 只要有中断字样就乱选，不看实际数据搬运方式。
- 把查询和中断混成“反正 CPU 都会参与”。
- 不知道 DMA 的重点是“直接搬大块数据”。

## 注意点

- 小白先抓关键词：`轮询`、`中断请求`、`大块直接传输`。
- 这是章节里最容易拿分的概念题。
