# 怎么判断 8086 寻址方式

## 这类题在考什么

这类题通常考：

```text
给你一个操作数表达式
判断它属于哪种寻址方式
```

它通常和默认段、物理地址一起连着考。

## 先记公式

最常见几类先记住：

```text
立即寻址
寄存器寻址
直接寻址
寄存器间接寻址
寄存器相对寻址
基址变址寻址
基址变址相对寻址
```

## 做题步骤

### 第一步：先看操作数是不是常数

如果是：

```asm
MOV AX, 1234H
```

那是立即寻址。

### 第二步：如果操作数直接是寄存器

比如：

```asm
MOV AX, BX
```

那是寄存器寻址。

### 第三步：如果中括号里只有直接地址

比如：

```asm
MOV AX, [1000H]
```

那是直接寻址。

### 第四步：如果中括号里只有一个地址寄存器

比如：

- `[BX]`
- `[SI]`
- `[DI]`
- `[BP]`

这是寄存器间接寻址。

### 第五步：如果“寄存器 + 位移”

比如：

```asm
[BX+10H]
```

这是寄存器相对寻址。

### 第六步：如果“基址寄存器 + 变址寄存器”

比如：

```asm
[BX+SI]
```

这是基址变址寻址。

### 第七步：如果“基址寄存器 + 变址寄存器 + 位移”

比如：

```asm
[BX+SI+20H]
```

这是基址变址相对寻址。

## 最简单例子

如果题目给：

```asm
MOV AX, [BX+SI+20H]
```

那它属于：

```text
基址变址相对寻址
```

因为同时有：

- 基址寄存器
- 变址寄存器
- 位移量

## 易错点

- 只要有中括号就一律说“间接寻址”，太粗了。
- `[1000H]` 是直接寻址，不是寄存器间接寻址。
- “寄存器 + 位移”和“基址变址 + 位移”别混。

## 注意点

- 判断时别急着背名词，先拆结构。
- 一拆成“有没有寄存器、几个寄存器、有没有位移”，就不难了。
