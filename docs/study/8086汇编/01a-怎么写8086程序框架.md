# 怎么写 8086 程序框架

## 这类题在考什么

这类题通常考：

```text
会不会写出一个完整可运行的 8086 汇编程序框架
```

最常见问法是：

- 补全程序骨架
- 判断哪句该放在哪个段
- 判断 `END START`、`ASSUME`、`MOV DS,AX` 的作用

## 先记公式

8086 最基本程序骨架可以先背成：

```asm
DATA SEGMENT
    ; 数据
DATA ENDS

STACK SEGMENT
    ; 栈空间
STACK ENDS

CODE SEGMENT
    ASSUME CS:CODE, DS:DATA, SS:STACK
START:
    MOV AX, DATA
    MOV DS, AX

    ; 程序主体

    MOV AH, 4CH
    INT 21H
CODE ENDS
END START
```

## 做题步骤

### 第一步：先分清三个最常见的段

- 数据放 `DATA SEGMENT`
- 栈放 `STACK SEGMENT`
- 指令放 `CODE SEGMENT`

### 第二步：写 `ASSUME`，但别以为它会真的改寄存器

`ASSUME` 只是告诉汇编器：

```text
哪个段寄存器默认对应哪个段名
```

它不是执行指令。

### 第三步：真正初始化 `DS`

考试特别爱考这个点：

```asm
MOV AX, DATA
MOV DS, AX
```

不能直接写：

```asm
MOV DS, DATA
```

### 第四步：入口和结束写完整

- 程序入口常写 `START:`
- 结束常写 `END START`

如果是 DOS 程序，结束通常用：

```asm
MOV AH, 4CH
INT 21H
```

## 最简单例子

如果题目让你写一个最小可运行框架，  
最稳的答法就是：

```asm
DATA SEGMENT
NUM DB 12H
DATA ENDS

CODE SEGMENT
    ASSUME CS:CODE, DS:DATA
START:
    MOV AX, DATA
    MOV DS, AX

    MOV AH, 4CH
    INT 21H
CODE ENDS
END START
```

## 易错点

- 写了 `ASSUME` 但没初始化 `DS`。
- 把数据定义写进代码段。
- 结束时只写 `RET`，没按程序环境正确退出。

## 注意点

- 小白先把“分段 + `ASSUME` + `DS` 初始化 + 程序退出”这四件事记死。
- 这是整门 8086 汇编最基础的送分题。
