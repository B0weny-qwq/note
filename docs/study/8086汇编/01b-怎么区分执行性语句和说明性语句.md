# 怎么区分执行性语句和说明性语句

## 这类题在考什么

这类题通常考：

```text
哪句会被 CPU 执行
哪句只是给汇编器看的说明
```

它是概念题，但考试里很常见。

## 先记公式

最稳的区分规则就是：

```text
执行性语句 -> 会生成机器指令，CPU 真执行
说明性语句 -> 伪指令，不由 CPU 执行
```

## 做题步骤

### 第一步：先看它是不是操作指令

像：

- `MOV`
- `ADD`
- `CMP`
- `JMP`

这些都属于执行性语句。

### 第二步：再看它是不是在“组织程序”或“定义数据”

像：

- `SEGMENT`
- `ENDS`
- `DB`
- `DW`
- `ASSUME`
- `END`

这些都属于说明性语句。

### 第三步：想一句最简单判断法

```text
CPU 能不能直接执行它
```

能执行就是执行性语句，  
不能执行、多半只是给汇编器看的，就是说明性语句。

## 最简单例子

比如：

```asm
MOV AX, BX
```

这是：

```text
执行性语句
```

因为它会被翻译成机器指令。

而：

```asm
NUM DB 12H
```

这是：

```text
说明性语句
```

因为它是在定义数据，不是让 CPU 去“执行 DB”。

## 易错点

- `DB/DW` 不是执行指令。
- `ASSUME` 不会真的改段寄存器。
- 不要因为一行里有英文就都当“指令”。

## 注意点

- 这类题很多是选择题，核心就是分清“执行”还是“说明”。
- 只要记住“伪指令给汇编器看”，就不容易乱。
