# 怎么判断 DB 和 DW 该怎么用

## 这类题在考什么

这类题通常考：

```text
这个数据该定义成字节还是字
DB/DW/DUP 各自什么意思
```

它通常和变量定义题连着出。

## 先记公式

```text
DB -> 定义 1 字节数据
DW -> 定义 1 字数据（2 字节）
```

`DUP` 表示重复定义，例如：

```asm
BUF DB 16 DUP(0)
```

表示定义 16 个初值为 0 的字节。

## 做题步骤

### 第一步：先看数据宽度

如果题目中的量只需 8 位，  
优先 `DB`。

如果需要 16 位，  
优先 `DW`。

### 第二步：再看它是不是数组/缓冲区

如果是一批同类型数据，  
常用 `DUP` 来写。

### 第三步：想清楚下标步长

- `DB` 数组按 1 字节走
- `DW` 数组按 2 字节走

### 第四步：如果要清零，就别写 `?`

- `DUP(0)` 表示清零
- `DUP(?)` 表示未初始化

## 最简单例子

定义一个字节变量：

```asm
VAL DB 12H
```

定义一个字变量：

```asm
NUM DW 1234H
```

定义 8 字节缓冲区：

```asm
BUF DB 8 DUP(0)
```

## 易错点

- 把 `DW` 当成“两个独立字节”随便用。
- 认为 `DUP(?)` 等于清零。
- 数组元素位宽和访问指令位宽对不上。

## 注意点

- 数据定义错了，后面寻址、步长、内容解释都会跟着错。
- 小白先从“字节还是字”这一步判断，不要上来就背格式。
