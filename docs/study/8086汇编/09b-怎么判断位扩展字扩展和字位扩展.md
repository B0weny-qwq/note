# 怎么判断位扩展字扩展和字位扩展

## 判断入口

扩展题先比较两个量：

```text
目标字数 vs 单片字数
目标位数 vs 单片位数
```

公式：

```math
\text{芯片数}
=
\frac{\text{目标字数}}{\text{单片字数}}
\times
\frac{\text{目标位数}}{\text{单片位数}}
```

如果只有位数不够，是位扩展。

如果只有字数不够，是字扩展。

如果字数和位数都不够，是字位扩展。

## 位扩展怎么做

位扩展解决“每个地址单元位数不够”的问题。

PPT 例 5.2：用 8 片 $64\mathrm{K}\times1$ 芯片组成 $64\mathrm{K}\times8$。

芯片数量：

```math
\frac{64\mathrm{K}}{64\mathrm{K}}\times\frac{8}{1}=8
```

连接方法：

- 8 片的地址线全部并联。
- 8 片的读写控制线全部并联。
- 8 片的片选线由同一个片选信号控制。
- 每片的 I/O 分别接 `D0~D7`。

地址范围不变，数据宽度变宽。

考点：位扩展不增加地址范围，只增加每个单元的位数。

## 字扩展怎么做

字扩展解决“地址单元数量不够”的问题。

PPT 例 5.3：用 $16\mathrm{K}\times8$ 芯片组成 $64\mathrm{K}\times8$。

芯片数量：

```math
\frac{64\mathrm{K}}{16\mathrm{K}}\times\frac{8}{8}=4
```

单片 $16\mathrm{K}$ 需要：

```math
16\mathrm{K}=2^{14}
```

所以每片芯片地址脚接 `A13~A0`。

目标 $64\mathrm{K}$ 需要：

```math
64\mathrm{K}=2^{16}
```

高位 `A15~A14` 用 2-4 译码器产生 4 个片选。

若从 `0000H` 开始，范围为：

| 组号 | `A15 A14` | 地址范围 |
| --- | --- | --- |
| 0 | `00` | `0000H~3FFFH` |
| 1 | `01` | `4000H~7FFFH` |
| 2 | `10` | `8000H~BFFFH` |
| 3 | `11` | `C000H~FFFFH` |

考点：字扩展会增加地址范围，必须增加片选译码。

## 字位扩展怎么做

字位扩展同时解决“字数不够”和“位数不够”。

PPT 例 5.4：用 $1\mathrm{K}\times4$ 的 2114 组成 $4\mathrm{K}\times8$。

总芯片数量：

```math
\frac{4\mathrm{K}}{1\mathrm{K}}\times\frac{8}{4}=4\times2=8
```

第一步，先位扩展：

```text
2 片 2114 -> 1K×8
```

一片接低 4 位 `D3~D0`，另一片接高 4 位 `D7~D4`。

第二步，再字扩展：

```text
4 组 1K×8 -> 4K×8
```

连接方法：

- `A9~A0` 接所有 2114 的地址脚。
- `A11~A10` 送入 2-4 译码器。
- 译码器 4 个输出分别选 4 组。

若从 `0000H` 开始，范围为：

| 组号 | 地址范围 |
| --- | --- |
| 0 | `0000H~03FFH` |
| 1 | `0400H~07FFH` |
| 2 | `0800H~0BFFH` |
| 3 | `0C00H~0FFFH` |

考点：字位扩展先把位数补齐，再扩地址范围。顺序反了也能理解，但画图时先位扩展更清楚。

## 快速判断表

| 目标和单片比较 | 扩展方式 | 片选变化 |
| --- | --- | --- |
| 字数相同，位数不够 | 位扩展 | 片选通常相同 |
| 字数不够，位数相同 | 字扩展 | 需要译码选组 |
| 字数不够，位数也不够 | 字位扩展 | 每组先并位，再译码选组 |

## 易错点

- 只算总容量，不分字数和位数。
- 位扩展时错误地增加地址线。
- 字扩展时忘记给每组单独片选。
- 字位扩展题只算出芯片数，没有说明每组几片、共有几组。
