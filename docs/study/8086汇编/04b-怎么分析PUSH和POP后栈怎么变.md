# 怎么分析 PUSH 和 POP 后栈怎么变

## 这类题在考什么

这类题通常考：

```text
执行几条 PUSH/POP 后
SP 怎么变
栈里存了什么
```

这是 8086 里特别高频的计算题。

## 先记公式

`PUSH`：

```text
SP = SP - 2
[SS:SP] = 源操作数
```

`POP`：

```text
目标操作数 = [SS:SP]
SP = SP + 2
```

## 做题步骤

### 第一步：先记住 8086 栈按“字”操作

所以每次 `PUSH/POP`，  
`SP` 通常变化 2。

### 第二步：遇到 `PUSH` 先减后存

这是最常考的细节。

### 第三步：遇到 `POP` 先取后加

顺序不要反。

### 第四步：多条指令按时间顺序一条条画表

最稳的方法就是列：

- 当前 `SP`
- 栈顶内容
- 执行后 `SP`

## 最简单例子

如果：

```text
SS = 2000H
SP = 1000H
AX = 1234H
```

执行：

```asm
PUSH AX
```

则：

```text
SP 先变成 0FFEH
再把 AX 的内容压入 [SS:0FFEH]
```

所以新栈顶偏移地址是：

```text
0FFEH
```

## 易错点

- 以为 `PUSH` 是先存后减。
- 忘了 8086 栈以字为单位，`SP` 不是减 1。
- `POP` 顺序和 `PUSH` 恢复顺序写反。

## 注意点

- 小白做题时最好画一个小栈图，不要纯脑算。
- 栈题本质上就是“顺序 + SP 变化”。
