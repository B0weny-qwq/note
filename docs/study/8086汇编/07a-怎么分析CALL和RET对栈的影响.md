# 怎么分析 CALL 和 RET 对栈的影响

## 这类题在考什么

这类题通常考：

```text
CALL 做了什么
RET 做了什么
为什么子程序能返回
```

它本质上是栈题。

## 先记公式

`CALL` 的核心作用：

```text
把返回地址压栈
再跳到子程序入口
```

`RET` 的核心作用：

```text
从栈顶弹出返回地址
再跳回调用点后面
```

## 做题步骤

### 第一步：看到 `CALL` 就想到“压返回地址”

不是压普通数据，  
而是压“接下来该回到哪”。

### 第二步：看到 `RET` 就想到“弹返回地址”

它不是随便跳，  
而是按栈里的地址回去。

### 第三步：如果题目问 `SP` 怎么变，就按字操作分析

8086 常见近调用里，  
`SP` 先减，再存返回地址。

### 第四步：如果过程里还自己 `PUSH` 了寄存器，要一起算

这类题经常混在一起出。

## 最简单例子

如果：

```asm
CALL PROC1
```

则可理解成：

```text
先把 CALL 下一条指令地址压栈
再跳到 PROC1
```

而 `PROC1` 结尾写：

```asm
RET
```

表示：

```text
把刚才压栈的返回地址取出来
回到主程序
```

## 易错点

- 以为 `CALL` 只是普通跳转。
- 忘了子程序里自己 `PUSH` 的东西也会影响栈顶位置。
- 把 `RET` 当成“回到过程开头后面一行”这种模糊概念。

## 注意点

- 小白把它想成“自动保存返回位置”就够用了。
- 一旦题目同时给了 `CALL + PUSH + POP + RET`，一定按顺序完整算栈。
