# 怎么判断子程序要不要保护寄存器

## 这类题在考什么

这类题通常考：

```text
子程序里改了哪些寄存器
调用前后为什么值会变
需不需要先保存再恢复
```

## 先记公式

最常见保护写法：

```asm
PUSH BX
PUSH SI
; 子程序主体
POP SI
POP BX
RET
```

核心思想就是：

```text
先保存现场
用完再恢复现场
```

## 做题步骤

### 第一步：先看子程序内部改了哪些寄存器

只要你改了调用方后面还要用的寄存器，  
就应考虑保护。

### 第二步：把“临时寄存器”和“返回寄存器”分开

有些寄存器本来就拿来返回结果，  
那就不一定恢复。

### 第三步：保护时按顺序压栈

恢复时顺序要反过来。

### 第四步：所有返回路径都要保持一致

不能某条路恢复了，某条路没恢复。

## 最简单例子

如果子程序里要临时使用 `BX` 和 `SI`，  
但主程序后面还要用它们原值，  
那最稳写法是：

```asm
PUSH BX
PUSH SI
; body
POP SI
POP BX
RET
```

## 易错点

- 只 `PUSH` 不 `POP`。
- 恢复顺序和保存顺序一样。
- 以为“能算对结果就行”，忽略调用方现场被破坏。

## 注意点

- 小白先记一句：
  `谁会被临时改，后面又还要用，谁就该保护。`
- 现场保护题本质是程序是否“调用后还能接着正常跑”。
