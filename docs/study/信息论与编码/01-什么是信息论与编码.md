# 什么是信息论与编码

## 这门课到底在研究什么

信息论与编码主要研究三件事：

1. 信息有多大，怎么度量。
2. 信息怎么压缩得更短。
3. 信息怎么在有噪声的信道里更可靠地传过去。

这门课不是单纯研究“代码怎么写”，而是研究“消息、信号、信道、编码”之间的数量关系。

## 专业名词解释

`信息论`
: 研究信息的度量、传输极限和处理规律的理论。

`编码`
: 按一定规则把原始消息转换成另一种符号序列的方法。

`信源编码`
: 去掉冗余，让表示更短，目标是提高传输有效性。

`信道编码`
: 主动加入监督信息，目标是提高传输可靠性。

`加密编码`
: 让消息即使被截获也难以理解，目标是提高传输安全性。

## 一张课程总图

```text
信源 -> 信源编码 -> 信道编码 -> 信道 -> 信道译码 -> 信源译码 -> 信宿
```

从课程章节上看，大致对应：

- 第 2 章：信源和熵，回答“信息有多大”
- 第 3 章：信道和容量，回答“信道最多能传多快”
- 第 4 章：率失真，回答“允许失真时最低要多少码率”
- 第 5 章：信源编码，回答“怎么逼近压缩极限”
- 第 6 章：信道编码，回答“怎么逼近可靠传输极限”

## 详细例子

假设你要把一句话发给同学：

```text
“明天八点到实验室”
```

可以分成三步理解：

1. 这句话本身来自“信源”。
2. 如果这句话里有很多重复表达，可以先压缩，这对应信源编码。
3. 如果传输过程中可能听不清、记错、丢字，就要额外加入校验信息，这对应信道编码。

## 生活类比

搬家时有三种思路：

- 信源编码像“真空压缩袋”，目标是少占空间。
- 信道编码像“在玻璃杯外面加泡沫板”，目标是少摔坏。
- 加密编码像“把纸箱贴成普通杂物”，目标是不让别人看懂。

## 易错点

- 不要把“压缩”和“纠错”混成一件事。
- 不要把“信息论”理解成只讲抽象公式，它最后都要落到通信系统。

## 注意点

- 信息论强调极限。
- 编码理论强调怎么接近这个极限。
