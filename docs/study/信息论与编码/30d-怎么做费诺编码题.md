# 怎么做费诺编码题

## 这类题在考什么

这类题通常考：

```text
已知一组符号概率
按费诺编码方法构造码字
```

它最核心的能力不是算某个公式，  
而是：

```text
会不会分组
```

## 先记公式

费诺编码没有一个像香农编码那样直接给码长的单公式，  
但它的核心规则可以记成一句：

```text
每次把符号分成两组
让两组总概率尽量接近
```

然后：

- 一组前面加 `0`
- 一组前面加 `1`

## 做题步骤

### 第一步：按概率从大到小排序

不排序，后面很难分得清楚。

### 第二步：先做第一层分组

把所有符号分成两组，要求：

```text
两组总概率尽量接近
```

注意是概率和接近，  
不是符号个数相同。

### 第三步：给两组分别赋 0 和 1

比如：

- 左组加 `0`
- 右组加 `1`

### 第四步：对子组继续重复分组

每一组内部再继续做同样的事，  
直到每组只剩一个符号。

### 第五步：把每层得到的 0、1 连起来

从根到叶子的路径，  
就是最后的费诺码字。

## 最简单例子

若概率为：

```math
0.4,\ 0.3,\ 0.2,\ 0.1
```

第一层可以分成：

- `{0.4,0.3}` 总和为 $0.7$
- `{0.2,0.1}` 总和为 $0.3$

然后分别赋：

- 左组加 `0`
- 右组加 `1`

第二层继续分：

- $0.4 \to 00$
- $0.3 \to 01$
- $0.2 \to 10$
- $0.1 \to 11$

于是得到一组费诺码：

```text
x1 -> 00
x2 -> 01
x3 -> 10
x4 -> 11
```

## 易错点

- 费诺编码看的是概率和，不是看每组符号个数。
- 分组方式有时不唯一，但必须合理、可译。
- 第一层分错，后面通常整题都会跟着错。

## 注意点

- 费诺编码题最值钱的地方是第一刀怎么分。
- 如果两种分法差不多，优先选后续更容易继续平衡的那种。
