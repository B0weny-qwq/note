# 怎么理解离散序列信源熵题

## 这类题在考什么

这类题通常考：

```text
一个序列整体有多少信息
每个符号平均有多少信息
```

最常见的是让你区分：

- 总熵
- 熵率
- 有记忆和无记忆的差别

## 先记公式

如果是无记忆离散序列信源，常有：

```math
H(X_1,X_2,\ldots,X_n)=nH(X)
```

参数解释：

- $H(X_1,X_2,\ldots,X_n)$：长度为 $n$ 的整个序列总熵。
- $H(X)$：单个符号的熵。
- $n$：序列长度。

熵率常写成：

```math
H_{\infty}=\lim_{n\to\infty}\frac{1}{n}H(X_1,X_2,\ldots,X_n)
```

参数解释：

- $H_{\infty}$：每个符号平均带来的信息量，也叫熵率。

## 做题步骤

### 第一步：先判断序列是无记忆还是有记忆

如果每个符号彼此独立，  
那就是无记忆序列。

如果前后符号有关联，  
那就是有记忆序列。

### 第二步：分清题目在问“总熵”还是“平均每符号信息量”

- 问整个长度为 $n$ 的序列有多少信息，想到总熵
- 问平均每个符号有多少信息，想到熵率

### 第三步：无记忆题直接用“乘以 n”

这是最基础、最常见的入门题型。

### 第四步：有记忆题先判断趋势

如果前后有关联，  
后一个符号更容易猜，  
那平均每个符号的信息量通常会下降。

## 最简单例子

如果某无记忆信源单符号熵为：

```math
H(X)=2\text{ bit}
```

长度为 5 的序列总熵就是：

```math
H(X_1,X_2,X_3,X_4,X_5)=5\times2=10\text{ bit}
```

而它的熵率仍然是：

```math
2\text{ bit/符号}
```

## 易错点

- 序列越长，不代表每个符号平均信息量越大。
- 总熵和熵率不要混。
- 有记忆时，不能再简单写成 $nH(X)$。

## 注意点

- 小白做题时，先问自己：题目是在问“总量”还是“平均量”。
- 一看到“相关性”两个字，就要意识到平均每符号信息量可能下降。
