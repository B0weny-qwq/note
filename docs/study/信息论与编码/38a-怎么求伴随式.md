# 怎么求伴随式

## 这类题在考什么

伴随式题通常考的是：

```text
收到一个码字 r
它有没有错
如果有错，大概错在哪
```

本质上就是做一次矩阵检验。

## 先记公式

```math
S=rH^T
```

参数解释：

- $S$：伴随式。
- $r$：接收码字。
- $H$：校验矩阵。
- $H^T$：校验矩阵转置。

## 做题步骤

### 第一步：把收码写成向量

题目一般会给：

```text
r = [100110]
```

先把它看成行向量。

### 第二步：把校验矩阵转置

公式是：

```math
rH^T
```

不是 `rH` 直接乱乘。

### 第三步：按模 2 运算

这是最关键的地方。  
这里不是普通加法乘法，而是：

```text
1 + 1 = 0
```

因为是在模 2 下。

### 第四步：看结果

- 如果 `S=0`，说明可能无错
- 如果 `S≠0`，说明检测到差错

## 最简单例子

如果算出来：

```math
S=[0\ 0\ 0]
```

那就表示：

```text
这个收码通过了校验
```

如果算出来：

```math
S=[1\ 0\ 1]
```

那就表示：

```text
它对应某种错误模式
```

## 小白最容易理解的版本

伴随式可以直接理解成：

```text
错误指纹
```

你把收码拿去和校验规则对一遍，  
最后出来的那串结果，就是错误的“报码”。

## 易错点

- 记得用 `H^T`，不是直接 `H`。
- 记得是模 2 运算。
- `S=0` 不是绝对百分百无错，而是“校验下看起来合法”。

## 注意点

- 简单题很多只问“求伴随式并判断是否码字”，这时算出 `S` 就够了。
- 如果题目继续问纠错，再看伴随式和差错图案的对应关系。
