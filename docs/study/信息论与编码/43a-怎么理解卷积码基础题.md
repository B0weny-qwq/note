# 怎么理解卷积码基础题

## 这类题在考什么

这类题通常考：

```text
卷积码和分组码差在哪
什么叫状态
状态图和网格图各看什么
```

入门题通常先考理解，不会一上来就逼复杂译码。

## 先记公式

卷积码最核心的理解式不是一个单数值公式，  
而是一句话：

```text
当前输出不仅和当前输入有关
还和前面若干时刻的状态有关
```

如果编码器有 $m$ 个记忆单元，  
那状态数常见为：

```math
2^m
```

参数解释：

- $m$：记忆单元个数。

## 做题步骤

### 第一步：先和分组码做区分

- 分组码：一块一块独立编码
- 卷积码：连续输入、连续输出，有状态记忆

### 第二步：看到寄存器，就想到“状态”

寄存器里存的前面若干拍输入，  
共同决定当前状态。

### 第三步：看到状态图，先看“从哪到哪”

状态图最重要的是：

- 当前状态是什么
- 输入 0 或 1 后转到哪个新状态
- 产生什么输出

### 第四步：看到网格图，先看“时间展开路径”

网格图就是把状态图沿时间轴展开。  
后面很多译码方法都靠它做路径比较。

## 最简单例子

如果一个卷积编码器有 2 个记忆单元，  
那状态数通常是：

```math
2^2=4
```

这 4 个状态可以理解成寄存器里前两拍输入的不同组合。

所以题目如果问：

```text
为什么卷积码是有状态编码
```

最稳的答法是：

```text
因为当前输出不仅取决于当前输入
还取决于寄存器中保存的过去输入状态
```

## 易错点

- 卷积码不是线性分组码再拼起来。
- 状态图和网格图不是两个无关概念，网格图是按时间展开的状态图。
- 记忆单元越多，状态数越多。

## 注意点

- 小白先把“状态”和“路径”理解清楚，比急着背复杂算法更重要。
- 卷积码章节的基础题，通常考到这里已经够覆盖入门分了。
