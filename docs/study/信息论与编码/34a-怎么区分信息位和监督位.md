# 怎么区分信息位和监督位

## 这类题在考什么

这类题通常考：

```text
一个码字里哪部分是原消息
哪部分是为了检错纠错加进去的
```

它经常和码长、码率、汉明码放在一起考。

## 先记公式

```math
n=k+r
```

参数解释：

- $n$：码长，也就是整个码字的位数。
- $k$：信息位个数。
- $r$：监督位个数，也叫校验位或冗余位个数。

## 做题步骤

### 第一步：先看题目给的是总长度还是信息长度

常见给法有：

- 已知 $(n,k)$ 码
- 已知信息位有几位，监督位有几位

这时先把三者关系列出来：

```math
n=k+r
```

### 第二步：分清“真正承载消息的位”和“额外补的位”

- 原始消息那部分是信息位
- 为了满足校验关系额外加进去的是监督位

### 第三步：看到系统码，优先判断前后哪段是信息位

系统码常见形式是：

```text
码字中有一段直接就是原消息
另一段是按规则算出来的监督位
```

### 第四步：如果题目问码率，再往下代

```math
R=\frac{k}{n}
```

参数解释：

- $R$：码率。
- $k$：信息位数。
- $n$：码长。

## 最简单例子

如果某码是：

```math
(7,4)
```

那就表示：

- 码长 $n=7$
- 信息位 $k=4$

所以监督位个数：

```math
r=n-k=7-4=3
```

## 易错点

- 别把监督位当成“没用的位”，它们是为了可靠传输。
- 别把 $(n,k)$ 中的顺序写反。
- 监督位个数是 $r=n-k$，不是 $k-r$。

## 注意点

- 一看到 $(7,4)$、$(15,11)$ 这类写法，要立刻反应成“码长 + 信息位数”。
- 这是后面生成矩阵、校验矩阵、汉明码题的基础。
