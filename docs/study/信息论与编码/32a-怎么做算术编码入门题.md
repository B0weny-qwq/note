# 怎么做算术编码入门题

## 这类题在考什么

这类题通常考：

```text
给定符号概率和一个短序列
一步一步求出最终编码区间
```

入门题重点不是复杂实现，  
而是会不会做区间缩放。

## 先记公式

初始区间通常设为：

```math
[0,1)
```

如果当前区间是：

```math
[L,U)
```

区间宽度为：

```math
W=U-L
```

参数解释：

- $L$：当前区间左端点。
- $U$：当前区间右端点。
- $W$：当前区间宽度。

某个符号对应子区间时，  
就是在当前区间内部按概率比例再切分。

## 做题步骤

### 第一步：先写出初始区间

入门题默认：

```math
[0,1)
```

### 第二步：按符号概率把区间切开

比如：

- $A$ 概率 $0.6$
- $B$ 概率 $0.4$

那初始区间可分成：

- $A \to [0,0.6)$
- $B \to [0.6,1)$

### 第三步：读序列的第一个符号，保留对应子区间

如果第一个符号是 $A$，  
那新区间就变成：

```math
[0,0.6)
```

### 第四步：在新区间内部继续按同样比例再切

这一步是算术编码的核心。  
不是回到 `[0,1)` 重来，  
而是在当前小区间里继续切。

### 第五步：最后写出最终区间

最终只要取这个区间内任意一个数，  
就能代表整个符号序列。

## 最简单例子

若：

- $A$ 概率为 $0.6$
- $B$ 概率为 $0.4$

要求编码序列：

```text
AB
```

第一步：

```math
[0,1) \to A:[0,0.6),\ B:[0.6,1)
```

因为第一个符号是 $A$，  
所以新区间为：

```math
[0,0.6)
```

第二步在 `[0,0.6)` 内再切：

- $A$ 对应宽度 $0.6\times0.6=0.36$
- $B$ 对应宽度 $0.6\times0.4=0.24$

所以：

- $A \to [0,0.36)$
- $B \to [0.36,0.6)$

第二个符号是 $B$，  
所以最终区间是：

```math
[0.36,0.6)
```

## 易错点

- 每一步都要在“当前区间”里切，不是每次都从 `[0,1)` 开始。
- 宽度按概率比例缩放，不是直接做简单相加。
- 最终结果通常是一个区间，不是单个固定码字。

## 注意点

- 入门题最关键的是区间更新过程写清楚。
- 只要你每一步区间都写对，最后结果通常就稳了。
