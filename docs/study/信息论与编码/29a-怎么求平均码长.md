# 怎么求平均码长

## 这类题在考什么

平均码长题考的是：

```text
平均每个符号，实际用了多少位
```

不是最长码字多长，  
也不是某一个符号多长，  
而是“按概率加权后的平均长度”。

## 先记公式

```math
L=\sum_{i=1}^{n} p(a_i)K_i
```

参数解释：

- $L$：平均码长。
- $a_i$：第 $i$ 个信源符号。
- $p(a_i)$：符号 $a_i$ 的概率。
- $K_i$：该符号码字的长度。

## 做题步骤

### 第一步：先列概率

把每个符号概率写出来。

### 第二步：再列码长

不要直接看码字内容，先只看长度。  
例如：

```text
x1 -> 0      长度 1
x2 -> 10     长度 2
x3 -> 110    长度 3
```

### 第三步：做概率加权

也就是：

```text
概率 × 码长
然后全部加起来
```

## 最简单例子

假设：

```math
p(x_1)=0.5,\quad p(x_2)=0.3,\quad p(x_3)=0.2
```

码字分别是：

```text
x1 -> 0
x2 -> 10
x3 -> 11
```

所以码长分别是：

```math
K_1=1,\quad K_2=2,\quad K_3=2
```

于是：

```math
L=0.5\times1 + 0.3\times2 + 0.2\times2 = 1.5
```

## 这类题最容易错在哪

### 错法 1：把码字本身加起来

你要加的是“长度”，不是 `0`、`10`、`11` 这些码字串。

### 错法 2：没乘概率

平均码长一定是加权平均，不是简单平均。

### 错法 3：漏符号

有几个符号，就要算几项。

## 怎么检查结果

记一个常识：

```text
平均码长一定介于最短码长和最长码长之间
```

比如最短 1 位，最长 4 位，  
你算出平均码长 6 位，那一定错。

## 易错点

- 平均码长不是最长码长。
- 平均码长不是码字个数。
- 概率和要先确认等于 1。

## 注意点

- 题目只要给“概率 + 码字”，基本就能直接求平均码长。
- 做编码效率题前，通常第一步就是先求平均码长。
