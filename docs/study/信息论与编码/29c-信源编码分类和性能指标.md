# 信源编码分类和性能指标

## 这部分在考什么

大纲里“信源编码的分类和性能指标”通常考：

```text
编码方法怎么分类
怎样评价一个信源编码好不好
```

## 信源编码的目标

信源编码的核心目标是：

```text
在满足可译码要求的前提下，让平均码长尽量短
```

也就是减少信源冗余，提高传输或存储效率。

## 常见分类

### 按是否允许失真

`无失真信源编码`
: 译码后必须完全恢复原消息，例如文本压缩、程序文件压缩。

`限失真信源编码`
: 允许一定失真，例如图像、语音、视频压缩。

### 按码字长度

`等长编码`
: 每个信源符号码长相同，简单但可能浪费。

`变长编码`
: 高频符号短码，低频符号长码，通常更接近熵极限。

### 按译码要求

`唯一可译码`
: 任意码字串只能译成唯一的源符号序列。

`即时码`
: 读到一个码字末尾就能立即译出，不需要继续等待。

即时码一定是唯一可译码，但唯一可译码不一定是即时码。

### 按处理对象

`单符号编码`
: 每次对一个信源符号编码。

`扩展信源编码`
: 把多个符号组成一组再编码，通常能更接近熵极限。

## 性能指标

### 平均码长

```math
L=\sum_i p_i l_i
```

参数解释：

- $p_i$：第 $i$ 个符号概率。
- $l_i$：第 $i$ 个符号码长。

平均码长越短，编码通常越省。

### 编码效率

```math
\eta=\frac{H(X)}{L}
```

参数解释：

- $H(X)$：信源熵。
- $L$：平均码长。

编码效率越接近 1，说明越接近理论下限。

### 编码冗余度

```math
\gamma=1-\eta
```

冗余度越小，编码越紧凑。

### 码率

如果每个信源符号平均用 $L$ bit 表示，码率常可理解为：

```text
平均每个信源符号需要传多少 bit
```

具体题目里要注意单位。

## 常见编码方法对比

| 方法 | 核心思路 | 是否常用于构造题 |
| --- | --- | --- |
| 香农编码 | 根据概率先定码长，再用累积概率写码字 | 常考 |
| 费诺编码 | 按概率尽量均衡分组 | 常考对比 |
| 霍夫曼编码 | 反复合并最小概率，构造最优前缀码 | 最常考 |
| 算术编码 | 把整个序列映射到一个区间 | 常考概念 |

## 易错点

- 平均码长不是普通算术平均，要按概率加权。
- 编码效率不能大于 1。
- 即时码和唯一可译码不是同一个概念。
- 变长编码不一定天然可译，必须检查前缀或唯一可译性。

## 注意点

- 题目问“性能指标”，优先写平均码长、编码效率、冗余度。
- 题目问“最优前缀码”，优先想到霍夫曼编码。
- 题目问“理论下限”，优先想到熵。
