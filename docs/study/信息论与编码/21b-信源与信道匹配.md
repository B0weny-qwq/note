# 信源与信道匹配

## 这部分在考什么

信源与信道匹配考的是：

```text
信源产生信息的速度
能不能被信道可靠地传过去
```

它把第 2 章的信源熵和第 3 章的信道容量连在一起。

## 核心判断

最核心的关系是：

```math
R_s \le C
```

参数解释：

- $R_s$：信源输出的信息率。
- $C$：信道容量。

意思是：

```text
信源每单位时间产生的信息量
不能超过信道每单位时间可靠承载的信息量
```

如果单位是“每个信源符号”和“每次信道使用”，就要先把单位换一致。

## 无失真传输时

如果信源要无失真传输，常见理解是：

```math
H(X) \le C
```

更准确地说，是信源熵率不能超过信道容量。

如果：

```math
H(X)>C
```

那么无论怎么设计编码，都不可能长期可靠地把全部信息传过去。

## 有失真传输时

如果允许失真，就不一定要求传完全部熵。

此时要看率失真函数：

```math
R(D) \le C
```

意思是：

```text
在允许失真 D 的条件下，最低需要传的码率不能超过信道容量
```

## 为什么要先信源编码再信道编码

信源编码和信道编码的目标相反：

- 信源编码：去掉无用冗余，让码率下降
- 信道编码：加入受控冗余，让传输更可靠

匹配的思路是：

```text
先把信源压到接近理论下限
再用信道编码保护它通过有噪声信道
```

## 最简单例子

假设某信源熵率为：

```math
R_s=2\text{ bit/s}
```

某信道容量为：

```math
C=3\text{ bit/s}
```

则有：

```math
R_s<C
```

理论上可以通过合适编码实现可靠传输。

如果信道容量只有：

```math
C=1\text{ bit/s}
```

则无失真可靠传输不可能。

## 易错点

- 不能只比较数字，必须先看单位是否一致。
- 容量不是某次实际传输速率，而是可靠传输上限。
- 加信道编码会增加码长，所以更需要先做好信源编码。

## 注意点

- 无失真看信源熵率和信道容量。
- 有失真看 $R(D)$ 和信道容量。
- 这部分常作为综合概念题出现。
