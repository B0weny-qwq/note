# 怎么判断是不是线性分组码

## 这类题在考什么

这类题通常考：

```text
给你一组码字
问它是不是线性分组码
```

本质上考的是：

```text
这组码字在模 2 加法下是否封闭
```

## 先记公式

对线性分组码，任意两个码字 $c_1,c_2$ 应满足：

```math
c_1 \oplus c_2 \in C
```

参数解释：

- $c_1,c_2$：两个合法码字。
- $\oplus$：模 2 加法。
- $C$：该码字集合。

还要记住一个常见结论：

```math
0 \in C
```

也就是全零码字通常必须在码字集合里。

## 做题步骤

### 第一步：先看是不是分组码

所有码字长度必须一样。  
长度不一样，直接就不是分组码。

### 第二步：先看全零码字在不在

如果一组码字里连全零码字都没有，  
那通常就不是线性分组码。

### 第三步：任取两个码字做模 2 加法

比如：

```math
011 \oplus 101 = 110
```

如果结果还在集合里，说明这两个码字满足线性要求。

### 第四步：多验几组，确认封闭性

简单题一般验出一个反例就够了：

- 如果某次相加结果不在集合里，立即判不是
- 如果题目数据很小，可以全部验完

## 最简单例子

给定码字集合：

```text
000, 011, 101, 110
```

先看：

- 长度都相同
- 有全零码字 `000`

再验：

```math
011 \oplus 101 = 110
```

结果还在集合里。

继续验下去，如果都满足，就可以判断它是线性分组码。

## 易错点

- 不是看“长得整齐”就叫线性分组码，关键看模 2 封闭性。
- 普通加法和模 2 加法不要混。
- 只验一组成立，不代表整组都成立。

## 注意点

- 考试里最稳的判断标准就是“有码长一致、含全零、对模 2 加法封闭”。
- 如果题目给了生成矩阵或校验矩阵，一般默认它讨论的是线性分组码。
