# 怎么判断是不是即时码

## 这类题在考什么

这类题通常考：

```text
收到一个完整码字后
能不能立刻译出对应符号
```

本质上考的是：

```text
有没有码字是另一个码字的前缀
```

## 先记公式

即时码最核心的判定规则可以直接记成一句：

```text
任一码字都不是其他码字的前缀
```

如果满足这条，就可以立刻判为即时码。

## 做题步骤

### 第一步：把所有码字列出来

先不要急着下结论，  
把每个码字都单独看一遍。

### 第二步：逐个检查前缀关系

比如看：

- `0` 是不是 `01` 的前缀
- `10` 是不是 `101` 的前缀

只要出现这种关系，就不是即时码。

### 第三步：一旦发现一个前缀关系，立刻判否

即时码的判断很干脆：

```text
有前缀关系 -> 不是即时码
无前缀关系 -> 是即时码
```

### 第四步：会顺手推出唯一可译

如果它是即时码，  
那它一定唯一可译。

## 最简单例子

如果：

- $A \to 0$
- $B \to 10$
- $C \to 11$

那：

- `0` 不是 `10` 或 `11` 的前缀
- `10` 也不是 `11` 的前缀

所以它是：

```text
即时码
```

反过来，如果：

- $A \to 0$
- $B \to 01$

那 `0` 是 `01` 的前缀，  
所以它：

```text
不是即时码
```

## 易错点

- 即时码考的是“前缀”，不是考“长度一样不一样”。
- 不是唯一可译就一定即时，方向别反了。
- 只要有一组前缀冲突，就直接判不是。

## 注意点

- 这类题最快的方法就是前缀检查。
- 如果码字多、你看得乱，直接画码树通常更稳。
