# 平均码长与编码效率

## 平均码长

平均码长表示：平均每个源符号编码后要用多少码元。

记作：

```math
L
```

课件里的平均码长写成：

```math
L = \sum_{i=1}^{n} p(a_i)\, K_i
```

参数解释：

- $L$：平均码长。
- $a_i$：第 $i$ 个信源符号。
- $p(a_i)$：符号 $a_i$ 出现的概率。
- $K_i$：符号 $a_i$ 对应码字的码长。

## 编码效率

编码效率衡量编码有多接近理论极限。

常写成：

```math
\eta = \frac{H}{L}
```

如果是 $r$ 元码，课件也常写成：

```math
\eta = \frac{H(X)}{L \log_2 r}
```

参数解释：

- $\eta$：编码效率。
- $H(X)$：信源熵。
- $L$：平均码长。
- $r$：码元进制数；二进制时 $r=2$，上式退化为 $\eta = H/L$。

## 详细例子

如果某信源熵是 $2\ \text{bit/符号}$，某种编码平均码长是 $2.5\ \text{bit/符号}$：

```math
\eta = \frac{2}{2.5} = 0.8
```

说明这套编码达到了 80% 的理论效率。

## 生活类比

打包快递时：

- 理论上最紧凑能装 10 件
- 你现在实际只能装 8 件

那么你的装箱效率就是 80%。

## 易错点

- 平均码长不是最长码长。
- 编码效率不是“压缩比”原样照搬，它是和熵比出来的。

## 注意点

- 优秀编码的目标就是让 `L` 尽量接近 `H`。
