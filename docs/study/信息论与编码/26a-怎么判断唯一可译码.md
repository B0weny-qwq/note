# 怎么判断唯一可译码

## 这类题在考什么

这类题通常考：

```text
给你一组码字
问它能不能唯一还原原消息
```

简单题最常见的就是让你找：

```text
同一串码流能不能拆成两种不同消息
```

## 先记公式

唯一可译码没有一个像熵那样直接代数值的单公式，  
它的核心判定思想是：

```text
任意有限长码流
只能对应唯一一种源符号序列
```

也就是：

```text
不能出现一串比特既能拆成这种
又能拆成另一种
```

## 做题步骤

### 第一步：先看有没有明显歧义

如果某个码流能拆成两种不同方式，  
那它立刻就不是唯一可译码。

### 第二步：优先检查短码流

简单题里，最常见的歧义往往出现在短串上。  
比如长度 2、3、4 的码流就可能出现冲突。

### 第三步：把可疑的拆分写出来

不是只写“有歧义”，  
而是明确写出：

```text
同一串码流 = 方案 1 = 方案 2
```

这样得分最稳。

### 第四步：如果没有明显歧义，再看它是不是前缀码

如果它本身是即时码或前缀码，  
那一定唯一可译。

## 最简单例子

如果：

- $A \to 0$
- $B \to 01$
- $C \to 1$

那码流：

```text
01
```

可以拆成：

- $B$
- $A,C$

也就是：

```text
01 = B = AC
```

所以这组码：

```text
不是唯一可译码
```

## 易错点

- 不是“每个码字都不同”就一定唯一可译。
- 别只说“好像有歧义”，要把具体歧义码流写出来。
- 即时码一定唯一可译，但唯一可译不一定即时。

## 注意点

- 小白做题时，最稳的方法就是主动去找一个反例码流。
- 一旦找到一个码流能拆成两种消息，题目就结束了。
