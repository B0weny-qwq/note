# 差错控制编码基本原理

## 这部分在考什么

差错控制编码的基本原理可以概括成一句话：

```text
通过加入受控冗余，让接收端能够发现错误或纠正错误
```

考试常考：

- 差错控制系统分类
- 为什么加监督位能检错纠错
- 最小码距和检错纠错能力
- 伴随式为什么能定位错误

## 差错是什么

在二进制信道里，最常见差错是比特翻转：

```text
0 变成 1
1 变成 0
```

如果发送码字为 $c$，接收向量为 $r$，差错图案为 $e$，则常写成：

```math
r=c+e
```

这里的加法通常是在模 2 下进行。

## 为什么冗余能帮助纠错

如果只传原始信息位，接收端通常无法判断某一位是不是错了。

加入监督位后，合法码字只占全部可能向量的一部分。  
接收端拿到一个向量后，就可以检查它是不是落在合法集合里。

```text
不在合法码字集合里 -> 说明可能发生差错
```

如果不同错误会落到不同检查结果，还可以进一步定位错误。

## 最小码距

两个码字之间的汉明距离是它们不同位的个数。

码字集合里的最小码距记作：

```math
d_{\min}
```

它决定了码的基本检错和纠错能力。

## 检错能力

若要检测 $t$ 位错误，需要：

```math
d_{\min}\ge t+1
```

等价地，一个最小码距为 $d_{\min}$ 的码最多能检测：

```math
d_{\min}-1
```

位错误。

## 纠错能力

若要纠正 $t$ 位错误，需要：

```math
d_{\min}\ge 2t+1
```

等价地，一个最小码距为 $d_{\min}$ 的码最多能纠正：

```math
t=\left\lfloor\frac{d_{\min}-1}{2}\right\rfloor
```

位错误。

## 伴随式思想

线性分组码里，校验矩阵 $H$ 用来检查合法性：

```math
cH^T=0
```

接收向量 $r$ 的伴随式为：

```math
S=rH^T
```

如果：

```math
S=0
```

说明没有检测到错误。

如果：

```math
S\ne0
```

说明接收向量不满足校验关系。  
在可纠错范围内，伴随式可以和错误图案对应起来。

## 和 ARQ、FEC 的关系

`ARQ`
: 主要靠检错和重发。

`FEC`
: 发送端提前加冗余，接收端直接纠错。

`HEC`
: 先纠错，纠不过再请求重发。

## 易错点

- 检错能力和纠错能力不是一个公式。
- 最小码距越大，检错纠错能力越强，但冗余也通常越多。
- 伴随式为 0 不一定代表物理上绝对没错，只代表没有检测到错误。

## 注意点

- 题目问“基本原理”，重点写加冗余、合法码字集合、最小码距。
- 题目给 $d_{\min}$ 时，马上想到检错 $d_{\min}-1$、纠错 $\lfloor(d_{\min}-1)/2\rfloor$。
