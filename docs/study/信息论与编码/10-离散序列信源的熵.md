# 离散序列信源的熵

## 为什么序列信源单独讲

真实通信里很少只发一个孤立符号，更多是连续符号序列。  
所以要研究“一个序列整体有多少平均信息”。

## 无记忆离散序列

如果每个符号独立同分布，那么长度为 `n` 的序列总熵近似就是单符号熵乘以 `n`。

```math
H(X_1, X_2, \ldots, X_n) = nH(X)
```

参数解释：

- $X_1, X_2, \ldots, X_n$：序列中第 1 到第 $n$ 个符号。
- $H(X_1, X_2, \ldots, X_n)$：整个长度为 $n$ 的序列总熵。
- $H(X)$：单个符号的熵。
- $n$：序列长度。

## 有记忆离散序列

如果前后符号相关，后一个符号更容易预测，那么平均每个符号提供的信息会下降。

这就引出了熵率的概念。

## 熵率

熵率就是“每个符号平均带来的信息量”。

可以理解成：

```math
\text{熵率} = \frac{\text{总熵}}{\text{序列长度}}
```

更正式地可写成：

```math
H_{\infty} = \lim_{n \to \infty} \frac{1}{n} H(X_1, X_2, \ldots, X_n)
```

参数解释：

- $H_{\infty}$：离散序列信源的熵率。
- $H(X_1, X_2, \ldots, X_n)$：长度为 $n$ 的序列总熵。
- $\frac{1}{n}H(\cdot)$：平均到每个符号的信息量。

## 详细例子

中文文本中，“中”后面接“国”的概率远高于随机。  
说明序列里有结构和记忆，后一个符号不是完全独立的，所以整体平均信息量会比“每个字完全随机”更低。

## 生活类比

你猜歌歌词时，听到前半句，后半句更容易猜。  
这说明歌词序列有记忆性，也说明冗余存在。

## 易错点

- 序列越长，不代表“每个符号的信息量”越大。

## 注意点

- 信源存在相关性，才会出现可以利用的压缩空间。
