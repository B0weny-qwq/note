# 信道编码定理

## 这部分在考什么

信道编码定理通常考概念，不是让你证明。

它回答的问题是：

```text
有噪声信道里，到底能不能做到可靠传输
可靠传输的码率上限是多少
```

## 核心结论

如果信息传输率 $R$ 小于信道容量 $C$：

```math
R<C
```

则存在足够好的编码方法，使译码错误概率可以任意小。

如果：

```math
R>C
```

则不可能实现任意可靠的传输。

## 人话理解

信道容量 $C$ 像一条路长期稳定能通过的最大车流量。

- 车流量低于上限：通过合理调度，堵车概率可以压得很低
- 车流量超过上限：再怎么调度也会长期拥堵

在信息论里：

- $R<C$：理论上可以可靠传
- $R>C$：理论上不可能长期可靠传

## 定理说明了什么

它说明两件事：

1. 有噪声不等于不能可靠通信。
2. 可靠通信存在上限，这个上限就是信道容量。

所以信道编码的目标不是“消灭噪声”，而是：

```text
在码率不超过容量的前提下，用编码把错误概率压低
```

## 它没有直接说明什么

信道编码定理主要是存在性结论。

它不直接告诉你：

- 具体码怎么构造
- 多长码长最合适
- 译码复杂度是多少
- 工程中一定能低成本达到容量

这些问题属于具体编码理论和工程设计。

## 和信道容量的关系

信道容量定义为：

```math
C=\max_{p(x)} I(X;Y)
```

信道编码定理则告诉我们：

```text
C 不是单纯的数学最大值，
而是可靠通信码率的理论边界。
```

## 和信源编码定理的区别

`信源编码定理`
: 关注压缩下限，核心量是信源熵。

`信道编码定理`
: 关注可靠传输上限，核心量是信道容量。

一句话区分：

```text
信源编码定理说“至少要多少 bit 才能表示”
信道编码定理说“最多能以多大速率可靠传”
```

## 易错点

- 不是只要加了信道编码就一定可靠，码率还要低于容量。
- 信道容量不是某个具体编码的码率，而是理论上限。
- 定理说的是错误概率可以任意小，不是有限码长下绝对不出错。

## 注意点

- 概念题看到“可靠通信上限”，优先答信道容量。
- 看到“有噪声但仍可可靠通信”，优先想到信道编码定理。
