# 怎么做霍夫曼编码题

## 这类题在考什么

这类题通常考：

```text
已知一组符号概率
构造霍夫曼码
```

有时还会顺带问：

- 平均码长
- 编码效率
- 为什么它比别的方法短

## 先记公式

霍夫曼编码最关键的不是一个单公式，  
而是一套固定流程：

```text
先排序
再合并最小的两个概率
一直合并到只剩一个根节点
最后回填 0 和 1
```

平均码长如果题目要算，用：

```math
L=\sum_{i=1}^{n} p_i l_i
```

参数解释：

- $L$：平均码长。
- $p_i$：第 $i$ 个符号概率。
- $l_i$：第 $i$ 个符号码长。

## 做题步骤

### 第一步：按概率从小到大排序

霍夫曼编码和香农、费诺不一样，  
它最关键的是：

```text
每次先抓最小的两个
```

### 第二步：把最小的两个概率合并

比如有：

```math
0.1,\ 0.2,\ 0.3,\ 0.4
```

先把 `0.1` 和 `0.2` 合并成 `0.3`。

### 第三步：把新节点重新放回去排序

这一点特别容易漏。  
合并后不是一路机械往下写，  
而是要重新比较大小。

### 第四步：重复合并，直到只剩一个总节点

当最后只剩一个根节点时，  
树结构就搭完了。

### 第五步：从根往下给边标 0 和 1

左边写 `0`、右边写 `1`，  
或者反过来也行，只要整棵树前后一致即可。

### 第六步：从根走到每个叶子，写出码字

路径上的 0、1 串就是该符号的码字。

## 最简单例子

设四个符号概率为：

```math
0.4,\ 0.3,\ 0.2,\ 0.1
```

先合并：

```math
0.1+0.2=0.3
```

得到：

```math
0.3,\ 0.3,\ 0.4
```

再合并两个较小节点，  
最后得到整棵树。

一种可能的霍夫曼码是：

- $x_1 \to 0$
- $x_2 \to 10$
- $x_3 \to 110$
- $x_4 \to 111$

如果继续求平均码长：

```math
L=0.4\times1+0.3\times2+0.2\times3+0.1\times3=1.9
```

## 易错点

- 合并后要重新排序。
- 左右标 `0/1` 不唯一，但同一棵树里不能乱变。
- 构树时抓错最小概率，会导致整题全错。

## 注意点

- 霍夫曼题最稳的做法是把每一步合并过程都写出来。
