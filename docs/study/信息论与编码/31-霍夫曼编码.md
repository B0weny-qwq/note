# 霍夫曼编码

## 定义

霍夫曼编码是对单符号离散无记忆信源构造最优前缀码的经典方法。

它的核心结论是：

```text
在所有前缀码里，它能把平均码长做到最短
```

这也是它和香农编码、费诺编码最本质的区别。

## 先说人话版原理

霍夫曼编码的逻辑非常像整理行李：

```text
最少用的东西，放最里面
最常用的东西，放最外面
```

放到码树里就是：

- 大概率符号尽量靠近根
- 小概率符号允许更深

因为大概率符号会反复出现，  
它每多走一层，平均码长就会被反复拉长。  
而小概率符号本来就不常出现，放深一点损失没那么大。

所以霍夫曼不是在“瞎合并最小概率”，  
它是在主动把最长路径留给最不常见的符号。

## 构造步骤

1. 把所有符号按概率从小到大排序。
2. 取最小的两个概率合并成一个新节点。
3. 继续排序并重复。
4. 最后从树根回填 0/1 得到各符号码字。

如果只背一句流程，就背：

```text
先合并最小，再一路回推
```

## 为什么一定先合并最小概率

假设树的最深层只能放两个符号，  
那这两个最深的位置，最应该留给谁？

答案一定是：

```text
留给最不常见的两个符号
```

因为如果你把高概率符号放到最深层，  
它每出现一次都要多写一位甚至多写几位，长期下来特别亏。

这就是霍夫曼编码最重要的直觉来源。

## 详细例子

假设 4 个符号概率分别为：

```text
0.4, 0.3, 0.2, 0.1
```

### 第一步：先合并最小的两个

先合并 `0.1` 和 `0.2`，得到一个新节点 `0.3`。

这一步的意思不是“它俩真的变成一个符号”，  
而是先把它们捆成一组，表示：

```text
这两个低概率符号未来都会处在树的更深处
```

### 第二步：把新节点重新参与排序

此时概率集合就变成：

```text
0.4, 0.3, 0.3
```

再继续选最小的两个合并，直到只剩一个总根节点。

### 第三步：从根往回分配 0 和 1

树搭完以后，再从上往下给边标 0 和 1。  
每个符号从根走到自己的叶子，沿路经过的 0/1 串，就是它的码字。

高概率符号离根更近，码字更短。

这句话是结果，不是巧合。  
它正是前面“先合并最小概率”的直接后果。

一种可能得到的霍夫曼码是：

```text
x1 -> 0
x2 -> 10
x3 -> 110
x4 -> 1110
x5 -> 1111
```

对应平均码长：

```math
L = 0.4\times1 + 0.3\times2 + 0.2\times3 + 0.05\times4 + 0.05\times4 = 2.0
```

这一步最好和前两种方法直接对照：

- 香农编码：`L = 2.5`
- 费诺编码：`L = 2.1`
- 霍夫曼编码：`L = 2.0`

一旦你把这三个数摆在一起，  
“为什么霍夫曼最优”这件事就不再只是口头结论了。

## 为什么它重要

它能得到平均码长最短的前缀码，因此在教材和工程里都非常核心。

你可以把它理解成：

```text
同样都要求可即时译码
霍夫曼是在这类规则里最省的那个
```

## 为什么它比费诺更强

费诺编码是：

```text
尽量均衡分组
```

霍夫曼编码是：

```text
严格把最深位置留给最小概率符号
```

前者更像“分组经验很好”，  
后者更像“从平均码长公式直接逼出来的最优策略”。

所以费诺常常不错，  
但霍夫曼在单符号离散无记忆信源上能给出更强的保证。

也就是说，费诺编码是：

```text
讲道理地分
```

霍夫曼编码是：

```text
按平均码长公式真正压到底
```

## 生活类比

像商场给高频商品分配离门口更近的位置，低频商品放更远。  
常用的路径更短，总体平均行走距离最小。

再换一个更贴近编码的类比：

如果每天都要打很多次字：

- 常用字母应该用最顺手的按键
- 很少用的符号可以放远一点

霍夫曼编码做的就是这种“高频元素优先缩短路径”的安排。

## 易错点

- 霍夫曼编码结果不唯一，但平均码长可相同。
- 构树时没保持小概率优先，会直接错。
- 合并后得到的新节点还要重新排序，不能一路机械往下写。
- 题目给的是前缀码最优问题时，别误用香农编码步骤。

## 注意点

- 题目一旦要求具体码字，最好把树完整画出来。
- 如果题目问“为什么最优”，关键回答不是“因为课本这么说”，而是“因为最小概率符号应占据最深位置，从而整体平均码长最小”。
