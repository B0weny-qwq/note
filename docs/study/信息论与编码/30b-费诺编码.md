# 费诺编码

## 定义

费诺编码也是一种经典无失真信源编码方法。  
它属于概率匹配编码，核心做法不是先算码长，而是不断分组：

```text
每次把符号分成两组
让两组总概率尽量接近
```

然后给两组分别分配 `0` 和 `1`，再继续往下细分。

## 先说人话版原理

费诺编码最核心的直觉是：

```text
如果左右两边总概率差不多
那两边平均走的层数也会比较合理
```

换句话说，它不是从“单个符号多长”开始想，  
而是从“整堆符号怎么分层更均衡”开始想。

这和香农编码的区别很大：

- 香农编码：先看每个符号该分几位
- 费诺编码：先看整堆符号怎么切两半

## 费诺编码步骤

### 第一步：按概率从大到小排序

这一步和香农编码、霍夫曼编码一样。  
因为你总得先知道谁更常见，谁更少见。

### 第二步：把符号分成两组

关键要求不是“每组个数相同”，而是：

```text
每组总概率尽量接近
```

这句话很重要。  
比如一组 2 个符号，另一组 3 个符号都没关系，  
只要两组概率和差不多即可。

### 第三步：给两组分别赋 0 和 1

第一层分组完成后：

- 一组前面加 `0`
- 另一组前面加 `1`

这意味着：

```text
这一层你先决定“往左走还是往右走”
```

### 第四步：对子组继续重复

对每一组再做同样的事：

- 继续分成概率尽量接近的两堆
- 再各自加一位

一直分到每组只剩一个符号为止。

## 详细例子

还是看这组课件常见概率：

```math
p(x_1)=0.4,\quad p(x_2)=0.3,\quad p(x_3)=0.2,\quad p(x_4)=0.05,\quad p(x_5)=0.05
```

### 第一次分组怎么想

你不会先问“某个符号几位”。  
你先问：

```text
这 5 个符号怎么切成两堆
才能让左右概率尽量平衡？
```

比如：

- 一边可能是 `0.4 + 0.3`
- 另一边可能是 `0.2 + 0.05 + 0.05`

你会发现两边总和比较接近。  
这就说明第一层这样切通常比较合理。

比如第一层可以这样分：

```text
第 1 组: x1(0.4), x2(0.3)   总概率 0.7
第 2 组: x3(0.2), x4(0.05), x5(0.05)   总概率 0.3
```

虽然不是完全相等，但已经比较接近，而且后续很好继续切分。

### 接着对子组再分

比如概率较大的那一组里，还要继续切；  
另一组里也要继续切。

所以费诺编码的整个过程很像：

```text
先粗分，再细分，再更细分
```

### 把这组例子真正写成码字

一种典型分法如下：

```text
第一层:
{x1, x2} -> 0
{x3, x4, x5} -> 1

第二层:
x1 -> 00
x2 -> 01
x3 -> 10
{x4, x5} -> 11

第三层:
x4 -> 110
x5 -> 111
```

于是得到一组费诺码：

```text
x1 -> 00
x2 -> 01
x3 -> 10
x4 -> 110
x5 -> 111
```

这组码字的平均码长是：

```math
L = 0.4\times2 + 0.3\times2 + 0.2\times2 + 0.05\times3 + 0.05\times3 = 2.1
```

这个结果比香农编码的 `2.5` 更短，  
说明“先做均衡分组”确实能进一步压低平均码长。

## 为什么费诺编码“看起来很合理”

因为它抓住了一个很自然的直觉：

```text
如果每一层两边的“权重”尽量接近
那平均路径长度通常不会太差
```

这和做任务分组很像：

- 如果一边工作量特别大，另一边特别小，就不平衡
- 如果每层都尽量平均，整体就更协调

## 为什么费诺编码不保证最优

因为“每一步局部看起来最平衡”，不等于“全局平均码长最小”。

它的问题不在方向错，而在于：

```text
局部最像均衡
不一定等于全局最省
```

所以费诺编码常常不错，  
但在严格比较平均码长时，通常还是霍夫曼编码更强。

上面的同组例子就是直接证据：

- 香农编码：`L = 2.5`
- 费诺编码：`L = 2.1`
- 霍夫曼编码：还能更低

## 费诺编码的优点

- 原理直观，容易从码树角度理解
- 很适合讲清“概率越大越靠前”这件事
- 对概率分布比较均衡的信源，效果往往不错

## 费诺编码的缺点

- 分组方式不唯一
- 不保证平均码长最短
- 分组一多，人工操作容易分错

## 和香农编码最大的区别

你可以把两者这样分：

`香农编码`
: 先决定“每个符号多长”。

`费诺编码`
: 先决定“整堆符号怎么分”。

香农更像“按个人定规格”，  
费诺更像“先做团队分班”。

## 生活类比

像老师给全班安排宿舍：

- 不是先给每个人单独定房间大小
- 而是先把全班分成两层楼
- 再在每层楼里继续分房间

目标是让每层楼的总人数、总需求尽量接近。  
这就是费诺编码“先分组再细分”的味道。

## 易错点

- 费诺编码不是平均分“符号个数”，而是尽量平衡“概率总和”。
- 分组不唯一不代表错误，只要满足编码规则、结果可译即可。
- 费诺编码通常是即时码，但不保证像霍夫曼那样达到最短平均码长。

## 注意点

- 题目一旦说“按概率分组，使每组尽量接近”，基本就是费诺编码。
- 如果两种分法都差不多，先检查哪种分法后续更顺、更容易继续平衡。
- 费诺编码常和 [码树](H:/note/docs/study/信息论与编码/28-码树.md) 一起理解，效果最好。
- 真做题时，第一层分组往往决定后面大半结果，所以别急着下刀，先比较几种分法的概率和。
