# 怎么画码树判断前缀关系

## 这类题在考什么

这类题通常考：

```text
给你一组码字
让你画码树
再判断是不是即时码
```

它其实是在考你会不会把一串 0、1 变成树结构。

## 先记公式

码树没有单独的代数公式，  
但要先记一个最核心的结构规则：

```text
从根到某个叶节点的一条路径
就是一个码字
```

在二元码里通常约定：

- 左分支记 `0`
- 右分支记 `1`

## 做题步骤

### 第一步：先从根节点开始

根节点本身不代表码字，  
它只是所有路径的起点。

### 第二步：按每个码字的 0、1 依次往下走

比如码字 `10`：

- 先走 `1`
- 再走 `0`

走完后，把这个点标成对应符号的叶节点。

### 第三步：把所有码字都放到同一棵树里

如果前面已经有同样的路径，就共用前面那段。  
这正是码树能看出前缀关系的原因。

### 第四步：检查叶节点下面还有没有继续长分支

如果某个叶节点下面还继续长出分支，  
就说明：

```text
有一个码字是另一个码字的前缀
```

这时它就不是即时码。

## 最简单例子

如果：

- $A \to 0$
- $B \to 10$
- $C \to 11$

画树时：

- `A` 放在根的左分支叶节点
- `B` 放在右再左
- `C` 放在右再右

你会发现：

```text
每个码字都停在叶节点
叶节点下面没有继续生分支
```

所以它是即时码。

如果改成：

- $A \to 0$
- $B \to 01$

那 `A` 的节点下面还要继续长 `1`，  
这就说明 `0` 是 `01` 的前缀，  
因此不是即时码。

## 易错点

- 叶节点和中间节点不要混。
- 同一路径前半段可以共用，不需要重复画。
- 叶节点下面如果继续长，往往就是前缀冲突。

## 注意点

- 码树题画得清楚，比口头说前缀关系更容易拿分。
- 一旦树画出来，即时码、前缀关系、构造霍夫曼码都会更好做。
