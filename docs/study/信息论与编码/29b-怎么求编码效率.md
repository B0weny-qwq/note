# 怎么求编码效率

## 这类题在考什么

编码效率题考的是：

```text
这套编码离理论最优还有多远
```

本质上就是比较：

- 理论下限：熵
- 实际花费：平均码长

## 先记公式

二进制时最常用：

```math
\eta=\frac{H(X)}{L}
```

参数解释：

- $\eta$：编码效率。
- $H(X)$：信源熵。
- $L$：平均码长。

## 做题步骤

### 第一步：先求熵

如果题目没直接给熵，就先求：

```math
H(X)
```

### 第二步：再求平均码长

如果题目给了码字表，就按平均码长公式先算出 $L$。

### 第三步：最后再相除

```math
\eta=\frac{H(X)}{L}
```

## 最简单例子

如果题目给：

```math
H(X)=2,\quad L=2.5
```

则：

```math
\eta=\frac{2}{2.5}=0.8
```

也就是：

```text
编码效率 = 80%
```

## 怎么理解这个结果

这表示：

```text
理论上最理想只要 2 位
你现在平均用了 2.5 位
```

所以这套编码已经比较接近理论极限，但还没压到最优。

## 怎么判断答案合理

正常情况下：

```text
编码效率不会超过 1
```

如果你算出：

```text
η = 1.2
```

那肯定有地方错了，通常是：

- 熵算错
- 平均码长算错
- 分母分子写反

## 易错点

- 别把 `L/H` 写成效率，效率是 `H/L`。
- 百分数和小数形式别混掉，`0.8` 就是 `80%`。
- 编码效率高，不代表码字一定短；它是相对理论极限的比较。

## 注意点

- 简单题基本套路就是：
  先求熵，再求平均码长，最后求效率。
- 做霍夫曼编码、香农编码、费诺编码比较题时，编码效率通常是最后一问。
