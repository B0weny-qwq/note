# 香农编码

## 定义

香农编码是一种经典无失真信源编码方法。  
它根据符号概率先确定码长，再利用累积概率分配码字。

课件里的核心思想可以概括成两句：

- 按概率大小排序
- 按概率决定码长，按累积概率决定码字

## 先说人话版原理

香农编码的逻辑其实很朴素：

```text
谁出现得越频繁，越应该少写几位
谁出现得越少，写长一点也能接受
```

但它不是直接“拍脑袋给短码”。  
它会先问：

```text
这个符号大概要配几位才合理？
```

然后再问：

```text
既然长度定了，那具体这几位写成什么？
```

所以香农编码最像：

```text
先定房间面积，再安排房号
```

## 码长怎么算

课件里给出的码长条件是：

```math
-\log_2 p(x_i) \le K_i < -\log_2 p(x_i) + 1
```

参数解释：

- $x_i$：第 $i$ 个信源符号。
- $p(x_i)$：符号 $x_i$ 出现的概率。
- $K_i$：符号 $x_i$ 的码长。

这句话别死背，可以这么理解：

```text
概率越小，-log2 p(xi) 越大，所以需要更长的码
概率越大，-log2 p(xi) 越小，所以可以用更短的码
```

## 香农编码步骤

### 第一步：按概率从大到小排序

先把所有符号按概率大小排好。

原因很简单：  
高概率符号应该优先分配更短的码，先排好更不容易乱。

### 第二步：算每个符号的码长

用概率去决定每个符号大概要占几位。

这一阶段你得到的是：

```text
这个符号该多长
```

但还没有得到：

```text
这个符号具体写成哪串 0 和 1
```

### 第三步：求累积概率

所谓累积概率，就是把当前符号前面的概率都加起来。  
它相当于在区间上给每个符号找一个起点。

通俗点说：

```text
前面的人站到哪了
你就从哪开始排号
```

### 第四步：把累积概率写成二进制小数

这是香农编码最有辨识度的一步。  
把累积概率转成二进制小数后，截取前 $K_i$ 位，作为这个符号的码字。

也就是说：

```text
码长告诉你“截几位”
累积概率告诉你“从哪段开始截”
```

## 详细例子

考虑这组概率：

```math
p(x_1)=0.4,\quad p(x_2)=0.3,\quad p(x_3)=0.2,\quad p(x_4)=0.05,\quad p(x_5)=0.05
```

### 先看码长直觉

- `0.4` 和 `0.3` 概率较大，码长会比较短
- `0.05` 很小，码长会明显变长

这一步不用先急着精确写码字，只要先接受一个事实：

```text
香农编码先解决“长短是否合理”
```

对于这组概率，一组典型码长可以写成：

```math
K_1=2,\quad K_2=2,\quad K_3=3,\quad K_4=5,\quad K_5=5
```

这正好体现了：

- `0.4`、`0.3` 概率大，码较短
- `0.05` 很小，所以需要更长的码

### 再看为什么要用累积概率

如果你只知道每个符号该写几位，却不知道写成什么，就可能冲突。  
累积概率相当于给每个符号安排不同的起始位置。

所以它的作用不是“算概率本身”，而是：

```text
避免不同符号拿到相互冲突的码字
```

### 把这组例子真正写成码字

先求累积概率：

```math
F_1=0,\quad F_2=0.4,\quad F_3=0.7,\quad F_4=0.9,\quad F_5=0.95
```

再把这些累积概率写成二进制小数，并按各自码长截取前 $K_i$ 位，可以得到一组香农码：

```text
x1 -> 00
x2 -> 01
x3 -> 101
x4 -> 11100
x5 -> 11110
```

这组码字的平均码长是：

```math
L = 0.4\times2 + 0.3\times2 + 0.2\times3 + 0.05\times5 + 0.05\times5 = 2.5
```

这一步很关键，因为你能真正看到香农编码的结果特征：

- 规则很整齐
- 码字分配明确
- 但尾部小概率符号会被拉得比较长

## 为什么香农编码不一定最优

因为它的顺序是：

```text
先定码长，再拼码字
```

这样做当然系统、规整、好操作。  
但问题是，它不一定把整棵码树利用得最满。

课件里也强调了这一点：  
香农码很多时候没有占满所有树叶，所以通常不是最佳码。

你可以把它理解成：

```text
房间面积分配得差不多合理
但整栋楼的空间还没用到极致
```

上面的例子就很典型：

- `x_4` 和 `x_5` 都被拉到 5 位
- 平均码长达到 `2.5`

这说明香农编码虽然合理，但还有继续压缩的空间。

## 香农编码的优点

- 步骤固定，方法系统
- 码字分配规则明确
- 结果基本唯一
- 很适合教学和理解“概率决定码长”这个核心思想

## 香农编码的缺点

- 平均码长通常不是最短
- 先有码长再有码字，灵活性不如构树法
- 在很多题里，性能会输给霍夫曼编码

## 生活类比

像给不同频率使用的快捷键分配长度：

- 最常用命令给两步快捷键
- 次常用命令给三步快捷键
- 很少用的命令给五步快捷键

香农编码就是先决定“每个命令允许走几步”，再决定具体走哪条路径。

## 易错点

- 香农编码不是按概率直接四舍五入变成码长。
- 累积概率不是用来算码长的，而是用来生成具体码字的。
- 香农编码结果通常唯一，不像费诺码和霍夫曼码那样可能有多个合法结果。

## 注意点

- 题目一旦问“香农编码步骤”，顺序必须说全：
  排序、算码长、求累积概率、转二进制小数、截取码字。
- 题目如果继续问平均码长和编码效率，记得回到 [平均码长与编码效率](H:/note/docs/study/信息论与编码/29-平均码长与编码效率.md) 的公式去算。
- 真做题时，香农编码最常错的不是公式，而是“累积概率写对了，但截码位数截错了”。
