# I/O 接口、寻址与数据传送方式

## 背景

外设和 CPU 在速度、数据格式、电平和时序上都不一致，所以不能直接硬连。第 6 章第一部分先解决“为什么要接口”和“接口怎么传数据”。

## 接口的作用

接口电路主要完成：

- 速度匹配。
- 电平与驱动能力匹配。
- 数据格式转换。
- 地址译码和设备选择。
- 状态检测与握手控制。

## 端口与编址方式

8086 有两种常见 I/O 编址方式：

- 存储器映像编址：把外设寄存器当成内存单元访问。
- 独立 I/O 编址：使用独立端口地址空间，通过 `IN/OUT` 访问。

区别：

- 存储器映像能用普通存储器指令。
- 独立编址地址空间单独管理，更节省内存地址。

## IN / OUT 规则

端口访问的固定规则：

- `IN AL, port` / `IN AX, port`
- `OUT port, AL` / `OUT port, AX`
- 端口号超过 `FFH` 时要先送到 `DX`。

## 三种数据传送方式

### 程序控制方式

CPU 反复查询设备状态，再决定是否读写。

特点：

- 结构简单。
- CPU 忙等，效率低。

### 中断方式

设备准备好后发中断请求，CPU 暂停当前任务去服务。

特点：

- 比查询方式效率高。
- 适合中速外设。

### DMA 方式

由 DMA 控制器直接在内存和外设之间搬数据。

特点：

- 速度高。
- CPU 参与少。
- 适合大批量数据传送。

## 状态口、数据口、控制口

设计接口题时常把寄存器分成三类：

- 数据口：真正交换数据。
- 状态口：反映忙/空、就绪、错误等状态。
- 控制口：下发工作命令或模式字。

## 这一章的分析顺序

看到接口题时按这个顺序想：

1. 设备需要输入还是输出。
2. CPU 如何选中设备。
3. 读写时用哪条指令。
4. 需要查询、中断还是 DMA。
5. 有没有握手信号和时序约束。
