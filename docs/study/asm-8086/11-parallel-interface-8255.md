# 并行接口 8255

## 背景

8255A 是最常见的可编程并行接口芯片。重点不是背引脚图，而是弄清楚“三个端口怎么分组、控制字怎么写、不同方式适合什么场景”。

## 端口结构

8255 提供三个 8 位端口：

- `PA`
- `PB`
- `PC`

其中 `PC` 还能拆成高 4 位和低 4 位。

分组方式：

- A 组：`PA + PC7-PC4`
- B 组：`PB + PC3-PC0`

## 两类控制方式

### 模式设置控制字

用于定义各端口的输入/输出方向和工作方式。

### BSR 控制字

只对 `PC` 某一位进行置位/复位，常用于单独控制控制线。

## 三种工作方式

### 方式 0

基本输入输出方式。

特点：

- 无握手。
- 适合 LED、拨码开关、简单键盘等。

### 方式 1

选通信号输入输出方式。

特点：

- 有握手线。
- 适合打印机、A/D 等需要就绪同步的外设。

### 方式 2

双向总线方式。

特点：

- 只对 `PA` 所在 A 组有效。
- 适合双向并行数据交换。

## 控制字书写思路

写控制字时按顺序判断：

1. 是模式设置还是 BSR。
2. A 组用哪种方式。
3. B 组用哪种方式。
4. `PA`、`PB`、`PC` 各位输入还是输出。

## 常见应用

课程中的典型实验：

- 七段数码管显示。
- 小键盘输入。
- 打印机并口接口。
- 简单开关量输入输出。

## 易错点

- `PC` 的部分位可能被握手功能占用，不能再当普通 I/O 用。
- 方式 2 只对应 A 组，不是所有端口都能双向总线。
- 读取/输出地址时要先分清 `PA/PB/PC/控制口` 的端口地址。
