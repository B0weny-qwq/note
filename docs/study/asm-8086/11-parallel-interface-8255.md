# 8255 并行接口实战

## 背景

8255 的关键是“控制字写对、端口方向想清、握手线不混用”。只背模式定义不够，要能从需求反推配置。

## 端口与分组

### 怎么用

- A 组：`PA + PC7..PC4`
- B 组：`PB + PC3..PC0`
- `PC` 可做普通 I/O 或握手控制位

### 注释与命名建议

- 端口地址命名：
  `PPI_PORTA`、`PPI_PORTB`、`PPI_PORTC`、`PPI_CTRL`。
- 方向配置注释明确：
  `; PA output, PB input, PC upper output`

### 易错点

- 忘记 `PC` 部分位可能被握手逻辑占用。

### 注意点

- 画表格列出每一位用途再写代码，避免口头推导遗漏。

## 控制字写法

### 怎么用

配置步骤：

1. 先确定是模式设置字还是 BSR。
2. 再确定 A 组模式和方向。
3. 再确定 B 组模式和方向。
4. 写入控制口。

```asm
MOV AL, ctrl_word
MOV DX, PPI_CTRL
OUT DX, AL
```

### 注释与命名建议

- 控制字常量用名字：
  `CTRL_MODE0_LED_KEY`、`CTRL_MODE1_PRINTER`。
- 注释写位语义，不只写十六进制。

### 易错点

- 把 BSR 字当模式字写入，导致端口方向配置失效。

### 注意点

- 修改单个位尽量用 BSR，避免重写整个模式字扰动其它端口。

## 模式 0/1/2 选型

## 模式 0

### 怎么用

- 普通输入输出，无握手，适合 LED、开关、简单键盘。

### 易错点

- 用在需要严格握手的外设上，导致丢数据。

### 注意点

- 适合低速、时序宽松场景。

## 模式 1

### 怎么用

- 选通输入输出，带握手，适合打印机等同步要求较高设备。

### 易错点

- 忽略 `PC` 握手位状态，软件逻辑与硬件状态不同步。

### 注意点

- 先读状态再读写数据，时序顺序固定。

## 模式 2

### 怎么用

- 双向总线方式，仅 A 组支持。

### 易错点

- 误认为 B 组也可配置模式 2。

### 注意点

- 双向切换时必须确认方向控制时序。

## 典型代码片段

```asm
; 配置 8255: PA output, PB input, mode 0
MOV DX, PPI_CTRL
MOV AL, CTRL_MODE0_LED_KEY
OUT DX, AL

; 输出到 PA
MOV DX, PPI_PORTA
MOV AL, led_bitmap
OUT DX, AL

; 读取 PB
MOV DX, PPI_PORTB
IN  AL, DX
MOV key_state, AL
```

## 章末检查清单

- 控制字是否区分模式字和 BSR 字。
- A 组/B 组方向是否与电路一致。
- 访问流程是否按“状态 -> 数据”顺序执行。
- 模式 2 是否仅用于 A 组。
