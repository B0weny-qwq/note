# 存储器分类与芯片扩展

## 背景

第 5 章先讲“存储器是什么”，再讲“如何接到 8086 上”。考试和实验更常考后半部分：容量计算、位扩展、字扩展、片选和地址分配。

## 存储器分类

按功能可分为：

- 主存、辅存、Cache。
- RAM 与 ROM。
- 易失性存储器与非易失性存储器。

常见器件：

- `SRAM`：速度快、结构简单、容量较小，常用于 Cache 或小容量高速场景。
- `DRAM`：容量大、成本低，需要刷新，常用于主存。
- `EPROM`：紫外擦除，可反复编程。
- `EEPROM`：电可擦，适合保存参数。
- `Flash`：容量大、非易失，常见于 U 盘、SSD、嵌入式固件。

## 性能指标

重点指标：

- 存储容量。
- 存取时间。
- 存储周期。
- 带宽。
- 可靠性和掉电保持能力。

## SRAM 和 DRAM 的区别

对比时抓住四点：

- `SRAM` 不需要刷新，速度更高，成本更高。
- `DRAM` 需要刷新，容量更大，价格更低。
- `SRAM` 常用于高速缓冲。
- `DRAM` 常用于主存。

## ROM 家族

按可改写能力记忆：

- `Masked ROM`：出厂固化。
- `PROM`：一次性写入。
- `EPROM`：紫外擦除后重写。
- `EEPROM`：按字节或小块电擦除。
- `Flash`：按块擦除，容量大，应用最广。

## 存储器扩展

8086 接口设计常见三类扩展：

- 位扩展：解决数据位数不够的问题。
- 字扩展：适配 8086 16 位数据总线和奇偶字节体。
- 容量扩展：多片芯片拼更大地址空间。

## 位扩展和字扩展

典型判断：

- 芯片是 `8` 位宽，CPU 需要 `16` 位数据通路时，要做字扩展。
- 芯片容量够但位宽不够，要并联扩展数据位。
- 8086 访问字数据时要区分偶地址低字节和奇地址高字节。

## 片选和地址分配

形成片选信号的常见方式：

- 线选法。
- 全译码法。
- 部分译码法。

分析步骤：

1. 先算芯片需要几根地址线。
2. 再看高位地址如何参与片选。
3. 最后写出起始地址、终止地址和片选逻辑。

## 做题口诀

- 先算容量，再算位宽。
- 先定芯片内部寻址线，再定外部片选线。
- 8086 字扩展必须考虑 `A0` 和 `BHE#`。
- 地址范围一定要写出首末地址，不只写片选条件。
