# 8253 定时计数器实战

## 背景

8253 场景题的核心是三件事：模式选型、控制字正确、初值计算正确。任一项错误都会表现为“看似运行但频率/时序全错”。

## 通道与信号

### 怎么用

每个通道都有：

- `CLK`：计数时钟输入
- `GATE`：门控输入
- `OUT`：输出

一般流程：

1. 写控制字
2. 写初值（低字节/高字节）
3. 按模式观察 `OUT`

### 注释与命名建议

- 通道用途命名：
  `tmr0_sys_tick`、`tmr1_beep`、`tmr2_capture`。
- 控制字常量命名：
  `TMR0_MODE2_LH_BIN`。

### 易错点

- 控制字配置了通道 1，实际却往通道 0 数据口写初值。

### 注意点

- 代码里把“控制字写入”和“目标通道初值写入”放在一起，避免跨段误配。

## 控制字写法

### 怎么用

控制字应明确：

- 计数器选择
- 读写格式（低字节、高字节、先低后高）
- 模式号
- BCD/二进制

```asm
MOV DX, TMR_CTRL_PORT
MOV AL, TMR0_MODE2_LH_BIN
OUT DX, AL
```

### 注释与命名建议

- 注释拆分控制字含义，例如：
  `; ch0 | L/H | mode2 | binary`

### 易错点

- 读写格式和后续写初值顺序不一致。

### 注意点

- 统一采用“先低后高”可减少歧义，除非题目要求其他格式。

## 初值计算

### 怎么用

常用关系：

```text
T = N / fclk
fout = fclk / N
```

反推初值：

```text
N = T * fclk
N = fclk / fout
```

### 注释与命名建议

- 计算过程注释写单位（Hz、ms）。
- 初值常量命名：`TMR0_RELOAD_1MS`。

### 易错点

- 单位混用（kHz、MHz、ms、us 混在一起）。
- `N` 超出 16 位范围没做分频方案调整。

### 注意点

- 先做数量级检查，再写寄存器值。

## 六种模式实用理解

## 模式 0

- 终端计数输出，适合“到时置位”。

## 模式 1

- 可重触发单稳态。

## 模式 2

- 分频脉冲输出，适合周期节拍。

## 模式 3

- 方波发生器，适合驱动蜂鸣器/时钟输出。

## 模式 4

- 软件触发选通脉冲。

## 模式 5

- 硬件触发选通脉冲。

### 易错点

- 模式 2 与模式 3 混用，导致波形占空比与预期不符。

### 注意点

- 先画波形预期，再选模式。

## 初始化代码模板

```asm
; tmr0: 1kHz tick from 1MHz clock -> N=1000
MOV DX, TMR_CTRL_PORT
MOV AL, TMR0_MODE2_LH_BIN
OUT DX, AL

MOV DX, TMR0_PORT
MOV AL, 0E8H           ; low byte of 1000
OUT DX, AL
MOV AL, 03H            ; high byte of 1000
OUT DX, AL
```

## 章末检查清单

- 控制字是否正确匹配目标通道。
- 初值写入顺序是否与读写格式一致。
- 初值计算是否含单位校验。
- 模式选择是否与目标波形/功能一致。
