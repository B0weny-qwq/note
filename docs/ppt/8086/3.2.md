# 8086 数据传送指令总结

## 0. 操作数符号速记

课件里常用这些缩写表示操作数类型：

```asm
i8      8位立即数
i16     16位立即数
imm     立即数，可能是 i8 或 i16

r8      8位通用寄存器：AH AL BH BL CH CL DH DL
r16     16位通用寄存器：AX BX CX DX SI DI BP SP
reg     通用寄存器，可能是 r8 或 r16

seg     段寄存器：CS DS ES SS

m8      8位存储器操作数
m16     16位存储器操作数
mem     存储器操作数，可能是 m8 或 m16

dest    目的操作数
src     源操作数
```

---

# 一、数据传送指令总览

8086 数据传送指令的作用：

```text
把数据从一个位置送到另一个位置
```

常见数据来源和目的地：

```text
寄存器
存储器
立即数
段寄存器
I/O 端口
堆栈
标志寄存器
```

这一类指令重点掌握：

```asm
MOV
PUSH
POP
XCHG
XLAT
IN
OUT
LEA
LDS
LES
LAHF
SAHF
PUSHF
POPF
```

注意：

```text
除 SAHF 和 POPF 等标志传送指令外，大多数数据传送指令不影响标志位。
```

---

# 二、MOV 传送指令

## 1. 基本格式

```asm
MOV dest, src
```

功能：

```asm
dest = src
```

也就是把源操作数送到目的操作数。

例：

```asm
MOV AL, 'B'       ; AL = 字符 B 的 ASCII 码
MOV AX, BX        ; AX = BX
MOV AL, [BX]      ; AL = DS:[BX]
MOV [BX], AL      ; DS:[BX] = AL
```

---

## 2. 初始化 DS 的常用写法

段寄存器不能直接接收立即数，所以不能这样写：

```asm
MOV DS, DATA      ; 错
```

正确写法：

```asm
MOV AX, DATA
MOV DS, AX
```

因为 DATA 是一个 16 位段地址，必须先送到普通寄存器，再送到 DS。

---

## 3. MOV 的常见合法形式

### 立即数送寄存器

```asm
MOV CL, 4
MOV DX, 0FFH
MOV SI, 200H
```

注意：

```text
以字母开头的十六进制数前面要加 0
```

比如：

```asm
MOV DX, 0FFH      ; 对
MOV DX, FFH       ; 有些汇编器可能不认
```

---

### 立即数送内存

```asm
MOV bvar, 0AH     ; bvar 是字节变量
MOV wvar, 0BH     ; wvar 是字变量
```

如果内存大小不明确，要写：

```asm
MOV BYTE PTR [BX], 255
MOV WORD PTR [BX], 255
```

---

### 寄存器之间传送

```asm
MOV AH, AL
MOV AX, BX
MOV DS, AX
MOV ES, AX
```

---

### 寄存器和内存之间传送

```asm
MOV AL, [BX]
MOV [BX], AL
MOV DX, [BP]
MOV DX, [BP+4]
MOV ES, [SI]
```

注意默认段寄存器：

```asm
[BX]、[SI]、[DI] 默认用 DS
[BP] 默认用 SS
```

例：

```asm
MOV AL, [BX]      ; AL = DS:[BX]
MOV DX, [BP]      ; DX = SS:[BP]
```

---

## 4. MOV 的非法情况

### 1）两个操作数类型不一致

```asm
MOV AL, 050AH     ; 错，AL 是 8 位，050AH 是 16 位
```

改成：

```asm
MOV AX, 050AH
```

或者：

```asm
MOV AL, 0AH
```

另一个例子：

```asm
MOV SI, DL        ; 错，SI 是 16 位，DL 是 8 位
```

---

### 2）两个操作数都是内存

```asm
MOV buf2, buf1    ; 错
```

8086 普通 MOV 不允许内存到内存。

正确写法：

```asm
MOV AX, buf1
MOV buf2, AX
```

如果是字节变量：

```asm
MOV AL, buf1
MOV buf2, AL
```

---

### 3）无法判断是字节还是字

```asm
MOV [BX+SI], 255  ; 错，不知道 [BX+SI] 是字节还是字
```

正确写法：

```asm
MOV BYTE PTR [BX+SI], 255
MOV WORD PTR [BX+SI], 255
```

---

### 4）段寄存器操作有限制

错误：

```asm
MOV DS, ES        ; 错
MOV DS, 100H      ; 错
MOV CS, [SI]      ; 错
```

正确：

```asm
MOV AX, ES
MOV DS, AX
```

或者：

```asm
MOV AX, 100H
MOV DS, AX
```

注意：

```text
CS 不能作为 MOV 的目的操作数。
```

---

# 三、数据段与变量定义

## 1. 数据段格式

```asm
DATA SEGMENT
AREA1  DB 14H, 3BH
AREA2  DB 3 DUP(0)
ARRAY  DW 3100H, 01A6H
STRING DB 'GOOD'
DATA ENDS
```

含义：

```asm
DB      Define Byte，定义字节
DW      Define Word，定义字
DUP     重复定义
```

例：

```asm
AREA2 DB 3 DUP(0)
```

表示：

```text
保留 3 个字节，初值都为 0
```

---

## 2. DW 的小端存储

8086 是小端方式：

```asm
ARRAY DW 3100H
```

内存中低地址放低字节：

```text
00H
31H
```

即：

```text
低字节在前，高字节在后
```

---

## 3. OFFSET 操作符

```asm
MOV DX, OFFSET ARRAY
```

功能：

```text
把 ARRAY 的偏移地址送到 DX
```

注意区别：

```asm
MOV AX, ARRAY          ; 取 ARRAY 里面的内容
MOV AX, OFFSET ARRAY   ; 取 ARRAY 的地址
```

---

# 四、堆栈指令 PUSH / POP

## 1. 堆栈基本特点

堆栈在 SS 段中，栈顶由 SP 指向。

特点：

```text
先进后出 FILO
栈操作单位是字，也就是 16 位
PUSH 只压入字
POP 只弹出字
SP 指向当前栈顶
入栈时 SP 减小
出栈时 SP 增大
```

栈的方向：

```text
高地址 -> 低地址 增长
```

---

## 2. PUSH 入栈

格式：

```asm
PUSH src
```

功能：

```asm
SP = SP - 2
SS:[SP] = src
```

src 可以是：

```text
16位通用寄存器
段寄存器
16位存储器操作数
```

不能是：

```text
立即数
8位寄存器
```

例：

```asm
PUSH AX
PUSH BX
PUSH DS
PUSH WORD PTR [BX]
```

错误：

```asm
PUSH AL       ; 错，不能压入字节
PUSH 1234H    ; 8086 下通常不允许立即数直接入栈
```

---

## 3. POP 出栈

格式：

```asm
POP dest
```

功能：

```asm
dest = SS:[SP]
SP = SP + 2
```

dest 可以是：

```text
16位通用寄存器
段寄存器
16位存储器操作数
```

但不能是：

```text
CS
8位寄存器
```

例：

```asm
POP AX
POP BX
POP DS
POP WORD PTR [SI]
```

错误：

```asm
POP DL       ; 错，DL 是 8 位
POP CS       ; 错
```

---

## 4. 堆栈例子

已知：

```asm
AX = 1000H
BX = 2000H
CX = 4000H
SP = 1000H
```

执行：

```asm
PUSH AX
PUSH BX
PUSH CX
POP BX
POP AX
POP CX
```

因为堆栈先进后出，所以最后：

```asm
BX = 4000H
AX = 2000H
CX = 1000H
```

---

# 五、XCHG 交换指令

## 1. 基本格式

```asm
XCHG dest, src
```

功能：

```text
dest 和 src 交换内容
```

例：

```asm
XCHG AX, BX
XCHG AH, AL
XCHG AX, WVAR
XCHG AL, BYTE PTR WVAR+1
```

---

## 2. XCHG 限制

可以交换：

```text
寄存器 <-> 寄存器
寄存器 <-> 内存
```

不可以：

```text
内存 <-> 内存
段寄存器参与交换
```

错误：

```asm
XCHG buf1, buf2   ; 错，两个都是内存
XCHG AX, DS       ; 错，段寄存器不能参与
```

---

# 六、XLAT 查表转换指令

## 1. 基本格式

```asm
XLAT
```

或者：

```asm
XLAT TABLE
```

功能：

```asm
AL = DS:[BX + AL]
```

使用前要准备：

```asm
BX = 表格首地址
AL = 表内偏移量
```

XLAT 查到的结果仍然放入 AL。

---

## 2. 典型例子

假设表从 0040H 开始：

```text
0040H: 30H
0041H: 31H
0042H: 32H
0043H: 33H
```

执行：

```asm
MOV BX, 0040H
MOV AL, 3
XLAT
```

结果：

```asm
AL = 33H
```

因为：

```asm
AL = DS:[BX + AL] = DS:[0040H + 3] = DS:[0043H]
```

---

## 3. 七段码查表示例

```asm
TABLE DB 40H, 79H, 24H, 30H, 19H
      DB 12H, 02H, 78H, 00H, 18H

MOV AL, 5
MOV BX, OFFSET TABLE
XLAT
```

结果：

```asm
AL = 12H
```

表示数字 5 对应的七段码。

---

# 七、I/O 输入输出指令 IN / OUT

8086 有独立的 I/O 端口地址空间。

---

## 1. IN 输入指令

格式一：端口号直接写在指令里。

```asm
IN AL, port8
IN AX, port8
```

例：

```asm
IN AL, 0F1H
IN AX, 80H
```

注意：

```text
直接端口地址只能是 8 位，也就是 00H ~ FFH。
```

格式二：端口号放在 DX 中。

```asm
IN AL, DX
IN AX, DX
```

例：

```asm
MOV DX, 310H
IN AL, DX
```

当端口号大于 FFH 时，必须用 DX。

---

## 2. OUT 输出指令

格式一：端口号直接写在指令里。

```asm
OUT port8, AL
OUT port8, AX
```

例：

```asm
OUT 85H, AL
OUT 44H, AX
```

格式二：端口号放在 DX 中。

```asm
OUT DX, AL
OUT DX, AX
```

例：

```asm
MOV DX, 0FF4H
OUT DX, AL
```

---

## 3. IN / OUT 注意点

累加器固定：

```text
8 位端口数据必须用 AL
16 位端口数据必须用 AX
```

不能随便用 BX、CX、DX 当数据寄存器。

错误：

```asm
IN BX, DX       ; 错，数据寄存器只能是 AL 或 AX
```

错误：

```asm
OUT 0FFEH, AL   ; 错，直接端口号超过 FFH
```

正确：

```asm
MOV DX, 0FFEH
OUT DX, AL
```

---

# 八、地址传送指令 LEA / LDS / LES

这类指令传送的是地址，不是普通数据内容。

---

## 1. LEA 取有效地址

格式：

```asm
LEA reg16, mem
```

功能：

```asm
reg16 = mem 的偏移地址
```

注意：

```text
源操作数必须是内存寻址形式
目的操作数必须是 16 位通用寄存器
不能是段寄存器
```

---

## 2. LEA 和 MOV 的区别

假设：

```text
DS = 5000H
SI = 1000H
DS:[1000H] = 1234H
```

执行：

```asm
LEA BX, [SI]
```

结果：

```asm
BX = 1000H
```

因为 LEA 取的是地址。

执行：

```asm
MOV BX, [SI]
```

结果：

```asm
BX = 1234H
```

因为 MOV 取的是内容。

---

## 3. LEA 和 OFFSET 的关系

下面两条在效果上类似：

```asm
LEA BX, TABLE
MOV BX, OFFSET TABLE
```

都是把 TABLE 的偏移地址送入 BX。

---

## 4. LEA 例子

```asm
LEA AX, [2345H]           ; AX = 2345H
LEA BX, [BP+SI]           ; BX = BP + SI
LEA SP, [0430H]           ; SP = 0430H
LEA BX, [BX+SI+0062H]     ; BX = BX + SI + 0062H
LEA DI, [BX]              ; DI = BX
```

---

## 5. LDS 装入远指针到 DS

格式：

```asm
LDS reg16, mem
```

功能：

```asm
reg16 = mem 中前 2 字节
DS    = mem 中后 2 字节
```

也就是从内存中取一个 4 字节远指针：

```text
偏移地址 + 段地址
```

例：

```asm
LDS SI, [450H]
```

如果：

```text
DS:[450H] = F346H
DS:[452H] = 0A90H
```

执行后：

```asm
SI = F346H
DS = 0A90H
```

---

## 6. LES 装入远指针到 ES

格式：

```asm
LES reg16, mem
```

功能：

```asm
reg16 = mem 中前 2 字节
ES    = mem 中后 2 字节
```

常用于 DI：

```asm
LES DI, [BX]
```

如果：

```text
DS:[BX]   = 0300H
DS:[BX+2] = 0500H
```

执行后：

```asm
DI = 0300H
ES = 0500H
```

---

# 九、标志传送指令

## 1. LAHF：FLAGS 低字节送 AH

格式：

```asm
LAHF
```

功能：

```text
把 SF、ZF、AF、PF、CF 等状态标志送到 AH 对应位
```

常用于保存部分标志位。

---

## 2. SAHF：AH 送 FLAGS 低字节

格式：

```asm
SAHF
```

功能：

```text
把 AH 的相关位送回 FLAGS 中
```

注意：

```text
SAHF 会影响标志位。
```

---

## 3. PUSHF：标志寄存器入栈

格式：

```asm
PUSHF
```

功能：

```asm
SP = SP - 2
SS:[SP] = FLAGS
```

用于保存整个标志寄存器。

---

## 4. POPF：标志寄存器出栈

格式：

```asm
POPF
```

功能：

```asm
FLAGS = SS:[SP]
SP = SP + 2
```

注意：

```text
POPF 会改变标志寄存器。
```

---

# 十、标志位操作指令

这些指令直接修改某些标志位。

## 1. CF 进位标志操作

```asm
CLC     ; CF = 0，清进位标志
STC     ; CF = 1，置进位标志
CMC     ; CF = ~CF，进位标志取反
```

---

## 2. DF 方向标志操作

```asm
CLD     ; DF = 0，串操作地址自动增加
STD     ; DF = 1，串操作地址自动减少
```

DF 主要影响字符串处理指令。

---

## 3. IF 中断标志操作

```asm
CLI     ; IF = 0，关中断
STI     ; IF = 1，开中断
```

---

# 十一、寻址方式复习

课件开头复习了几种寻址方式。

已知：

```asm
DS = 2000H
ES = 2100H
SS = 1500H
SI = 00ADH
BX = 0100H
BP = 0010H
```

物理地址公式：

```text
物理地址 = 段地址 × 10H + 偏移地址
```

---

## 1. 直接寻址

```asm
MOV AX, [100H]
```

偏移地址：

```text
100H
```

默认段：

```text
DS
```

物理地址：

```text
2000H × 10H + 100H = 20100H
```

---

## 2. 寄存器间接寻址

```asm
MOV BX, [SI]
```

偏移地址：

```text
SI = 00ADH
```

默认段：

```text
DS
```

物理地址：

```text
200ADH
```

---

## 3. 寄存器相对寻址

```asm
MOV CX, [BX+10H]
```

偏移地址：

```text
BX + 10H = 0100H + 10H = 0110H
```

物理地址：

```text
20110H
```

---

## 4. 基址变址寻址

```asm
MOV AX, [BX][SI]
```

偏移地址：

```text
BX + SI = 0100H + 00ADH = 01ADH
```

物理地址：

```text
201ADH
```

---

## 5. BP 默认走 SS

```asm
MOV CL, [BP+4]
```

偏移地址：

```text
BP + 4 = 0010H + 4 = 0014H
```

默认段：

```text
SS
```

物理地址：

```text
15014H
```

---

## 6. 段超越前缀

```asm
MOV CH, ES:[BP+SI]
```

虽然用了 BP，但前面有 ES:，所以段地址用 ES。

偏移地址：

```text
BP + SI = 0010H + 00ADH = 00BDH
```

物理地址：

```text
2100H × 10H + 00BDH = 210BDH
```

---

# 十二、最容易错的点

## 1. MOV 不能内存到内存

错误：

```asm
MOV A, B
```

正确：

```asm
MOV AX, B
MOV A, AX
```

---

## 2. 段寄存器不能直接装立即数

错误：

```asm
MOV DS, 1000H
```

正确：

```asm
MOV AX, 1000H
MOV DS, AX
```

---

## 3. CS 不能随便改

错误：

```asm
MOV CS, AX
POP CS
```

CS 通常由跳转、调用、中断、返回等控制转移指令改变。

---

## 4. PUSH / POP 只能操作字

错误：

```asm
PUSH AL
POP DL
```

正确：

```asm
PUSH AX
POP DX
```

---

## 5. IN / OUT 只能用 AL 或 AX 传数据

错误：

```asm
IN BX, DX
OUT DX, BX
```

正确：

```asm
IN AL, DX
IN AX, DX
OUT DX, AL
OUT DX, AX
```

---

## 6. 端口号超过 FFH 必须用 DX

错误：

```asm
OUT 0FFEH, AL
```

正确：

```asm
MOV DX, 0FFEH
OUT DX, AL
```

---

## 7. LEA 取地址，MOV 取内容

```asm
LEA BX, [SI]     ; BX = SI 这个偏移地址
MOV BX, [SI]     ; BX = 内存 DS:[SI] 中的内容
```

这个很容易混。

---

## 8. XLAT 隐含使用 BX 和 AL

```asm
XLAT
```

实际等价于：

```asm
AL = DS:[BX + AL]
```

所以使用前必须准备好：

```asm
BX = 表首地址
AL = 表内偏移
```

---

# 十三、一句话总览

这部分的核心就是：

```text
MOV 负责普通数据搬运；
PUSH/POP 负责堆栈；
XCHG 负责交换；
XLAT 负责查表；
IN/OUT 负责端口输入输出；
LEA/LDS/LES 负责地址传送；
LAHF/SAHF/PUSHF/POPF 负责标志寄存器传送；
CLC/STC/CMC/CLD/STD/CLI/STI 负责直接操作标志位。
```
